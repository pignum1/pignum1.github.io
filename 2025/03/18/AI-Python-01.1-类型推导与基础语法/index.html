1. Python 与 Java 类型系统对比

特性 Python Java
类型检查时机 运行时（动态） 编译时（静态）
变量声明 无需声明类型 必须声明类型 int x = 10;
类型安全 运行时发现错误 编译时发现错误
灵活性 高（同一变量可赋不同类型） 低（类型一旦确定不能变）
IDE 支持 类型注解提供辅助 原生强类型支持
适用场景 快速原型、数据探索 大型项目、团队协作

总结

本文详细介绍了 Python 的类型推导与基础语法：

  1. Python vs Java 类型系统对比

    • Python 是动态类型，运行时检查
    • Java 是静态类型，编译时检查
    • Python 变量无需声明类型

  2. 基础数据类型

    • int: 整数，精度不限
    • float: 浮点数，双精度
    • complex: 复数（Java 无内置）
    • bool: 布尔值
    • None: 空值（对应 Java 的 null）

  3. 字符串操作

    • f-string 格式化
    • 丰富的字符串方法
    • 查找、替换、分割

  4. 集合类型

    • list: 可变有序列表
    • dict: 键值对字典
    • set: 唯一元素集合

  5. 类型注解 (Type Hints)

    • 提高代码可读性
    • IDE 自动补全支持
    • mypy 静态类型检查
