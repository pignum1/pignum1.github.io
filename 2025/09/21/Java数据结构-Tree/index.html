什么是树形结构？

在计算机科学中，**树（Tree）**是一种重要的非线性数据结构，它以分层的方式存储数据。树被用来表示具有层级关系的数据，例如文件系统中的目录结构、公司的组织架构图等。

一个树结构包含一系列带有父子关系的节点。每个节点都有一个父节点（除了顶部的根节点）和零个或多个子节点。

核心特性

  • 根节点（Root）: 树顶部的节点，唯一没有父节点的节点。
  • 子节点（Child）: 一个节点下面直接连接的节点。
  • 父节点（Parent）: 一个节点上面直接连接的节点。
  • 叶子节点（Leaf）: 没有任何子节点的节点。
  • 深度（Depth）: 从根节点到某个节点的路径长度。
  • 高度（Height）: 从某个节点到其最远叶子节点的路径长度。

什么是二叉树 (Binary Tree)？

二叉树是树形结构中最常用的一种特殊类型。它的核心特点是每个节点最多只能有两个子节点，这两个子节点通常被称为“左子节点”和“右子节点”。

二叉树的关键属性

  • 有序性: 二叉树的子节点是严格区分左右的。一个节点的左子树和右子树的顺序不能颠倒，即使某个节点只有一个子节点，也要明确它是左子节点还是右子节点。
  • 递归定义:
    1. 二叉树可以为空（没有节点）。
    2. 若不为空，则它由一个根节点以及两棵互不相交的、分别称为“左子树”和“右子树”的二叉树组成。

特殊的二叉树

在基础的二叉树之上，有两种非常重要的特殊形态：

  1. 满二叉树 (Full Binary Tree)
    • 定义: 除了叶子节点外，每个节点都有两个子节点。也就是说，不存在只有一个子节点的节点。
    • 特点: 结构非常规整，所有叶子节点都在同一层级。

二叉树的遍历

“遍历”是指按照某种特定的顺序访问树中的每一个节点，并且每个节点只被访问一次。这是二叉树最基本、最重要的操作之一。

遍历方法定义

  1. 前序遍历 (Pre-order)

    • 规则: 根 -> 左 -> 右
    • 遍历结果: A → B → D → E → C → F

  2. 中序遍历 (In-order)

    • 规则: 左 -> 根 -> 右
    • 遍历结果: D → B → E → A → F → C

  3. 后序遍历 (Post-order)

    • 规则: 左 -> 右 -> 根
    • 遍历结果: D → E → B → F → C → A

Java 实现二叉树遍历：递归与非递归

遍历方法深度解析

1. 前序遍历 (Pre-order: 根-左-右)
  • 递归实现:
    • 逻辑: 完美契合“定义-执行”的思维模式。代码即定义：先处理当前节点，然后递归调用左子树，再递归调用右子树。
    • 优势: 极其直观，代码量最少，可读性最高。
  • 非递归 (栈) 实现:
    • 逻辑: 用栈模拟函数调用。因为栈是“后进先出”，为了实现“先左后右”的访问顺序，入栈时必须先压右子节点，再压左子节点。
    • 对比: 这是最容易从递归转为非递归的遍历。核心就是用一个栈来管理待访问的节点，顺序控制非常直接。
2. 中序遍历 (In-order: 左-根-右)
  • 递归实现:
    • 逻辑: 先递归深入左子树，到底后处理节点，再递归右子树。
    • 优势: 同样直观，符合定义。
  • 非递归 (栈) 实现:
    • 逻辑: 这是三种遍历中，非递归实现最复杂的。它需要一个循环来将所有左子节点一路压栈，直到最左端。然后，从栈中弹出一个节点进行处理，并转向其右子树，对右子树重复相同的压栈过程。
    • 对比: 它不像前序遍历那样简单地“访问-压栈”，而是“一路向左压栈 -> 访问 -> 转向右子树”，需要一个指针来辅助移动。
3. 后序遍历 (Post-order: 左-右-根)
  • 递归实现:
    • 逻辑: 必须先完成左右子树的递归调用，最后才能处理根节点。
    • 优势: 逻辑清晰。
  • 非递归 (栈) 实现:
    • 逻辑 (双栈法): 这是最巧妙的实现之一。它利用了后序遍历 左-右-根 的逆序是 根-右-左 这一特性。
      1. 第一个栈的逻辑和前序遍历非常相似，但压栈顺序改为先左后右，这样出栈顺序就是 根-右-左。
      2. 将每次出栈的节点压入第二个栈。
      3. 最后，从第二个栈依次弹出所有元素，得到的序列就是 左-右-根。
    • 对比: 虽然使用了两个栈（或一个栈加一个Collections.reverse()），但它将一个复杂问题转换为了一个我们已经解决的问题（前序遍历的变种），是算法设计中“问题转换”思想的绝佳体现。

递归 vs. 非递归对比

特性 递归实现 非递归实现 (迭代)
代码简洁度 非常高，代码逻辑清晰，符合遍历的定义。 相对复杂，需要手动维护一个栈。
空间复杂度 O(H)，H 是树的高度。最坏情况是 O(N)。 O(H)，栈的大小。最坏情况是 O(N)。
性能 可能会因函数调用开销略慢，且易导致栈溢出。 通常性能更好，且没有栈溢出风险。
理解难度 容易理解概念。 需要理解如何用栈模拟递归过程，稍难。

平衡二叉搜索树 (Self-Balancing BST)

我们已经了解了二叉树，而二叉搜索树 (Binary Search Tree, BST) 是一种特殊的二叉树，它具有以下特性：

  • 对于任意节点，其左子树中的所有节点值都小于该节点的值。
  • 其右子树中的所有节点值都大于该节点的值。
  • 它的左右子树也都是二叉搜索树。

这个特性使得 BST 的查找效率非常高，平均时间复杂度为 O(log n)。但它有一个致命的弱点：在极端情况下，如果插入的元素是完全有序的（例如 1, 2, 3, 4, 5），BST 会退化成一个链表，查找时间复杂度也随之恶化为 O(n)。

    graph TD
    subgraph "退化的二叉搜索树 (链表)"
        A(1) --> B(2)
        B --> C(3)
        C --> D(4)
        D --> E(5)
    end

为了解决这个问题，平衡二叉搜索树 应运而生。它的核心思想是：在每次插入或删除节点后，通过一系列操作来保持树的平衡，使其高度尽可能低，从而始终保证 O(log n) 的时间复杂度。

实现平衡的关键，就是树的旋转。

核心平衡操作：树的旋转

树的旋转是重新调整树结构、降低树高度的局部操作，它在不改变树的中序遍历序列（即不破坏 BST 性质）的前提下，调整节点间的父子关系。旋转分为两种：左旋和右旋。

为了更好地理解，我们将通过分步图解来详细展示旋转过程。

1. 右旋 (Right Rotation)

AVL 树：绝对平衡的追求者

AVL 树是最早被发明的自平衡二叉搜索树。它追求绝对的平衡。

  • 平衡条件: 对于树中的任何一个节点，其左子树和右子树的高度差不能超过 1。这个高度差被称为“平衡因子”。
  • 实现方式: 在每次插入或删除后，从操作点开始向上回溯，检查每个节点的平衡因子。一旦发现某个节点的平衡因子大于 1，就立即通过单旋转或双旋转（例如先左旋再右旋）来恢复平衡。
  • 优缺点:
    • 优点: 查找性能极高，因为树的高度总是最低的。
    • 缺点: 为了维持这种严格的平衡，插入和删除时可能需要进行多次旋转，维护成本较高。

红黑树 (Red-Black Tree)：工程上的最佳选择

红黑树是另一种更常见的自平衡二叉搜索树，它在工程领域应用极广，Java 中的 TreeMap、TreeSet 以及 HashMap 的树化结构都基于它。

与 AVL 树追求“绝对平衡”不同，红黑树追求的是一种“大致的平衡”，它通过给每个节点赋予红色或黑色的属性，并遵循几条简单的规则来维持平衡。

  • 五大规则:

    1. 每个节点要么是红色，要么是黑色。
    2. 根节点永远是黑色的。
    3. 所有叶子节点（NIL 节点，空节点）都是黑色的。
    4. 不能有两个相邻的红色节点（即红色节点的父节点和子节点都不能是红色的）。
    5. 从任意一个节点到其所有后代叶子节点的路径上，包含的黑色节点数量必须相同。

  • 实现方式: 当插入或删除节点时，可能会破坏上述规则（最常见的是规则 4 和 5）。此时，红黑树通过变色和树旋转（最多两次旋转）来重新满足这些规则，从而恢复平衡。

  • 优缺点:

    • 优点: 插入和删除的效率比 AVL 树更高，因为它的平衡条件更宽松，需要旋转的次数更少。
    • 缺点: 查找性能略低于 AVL 树，因为它的树高可能不是严格最低的。但它仍然保证了最长路径不超过最短路径的两倍，确保了 O(log n) 的性能。

结论： 红黑树在查找、插入、删除等多种操作之间取得了更好的整体性能平衡，因此成为了工业界实现有序映射和集合的首选数据结构。
