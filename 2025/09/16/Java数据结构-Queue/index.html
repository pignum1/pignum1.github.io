Java 集合框架：Queue 接口详解

在 Java 集合框架中，Queue (队列) 接口是一种特殊的线性数据结构，它遵循 先进先出 (FIFO - First-In, First-Out) 的原则。这意味着最先进入队列的元素将最先被移出。队列在计算机科学中应用广泛，常用于模拟排队、任务调度、广度优先搜索等场景。

1. Queue 接口的核心方法

Queue 接口定义了一系列操作队列的方法，这些方法分为两组，其行为略有不同：一组在操作失败时会抛出异常，另一组则会返回特殊值（如 null 或 false）。

操作 抛出异常 返回特殊值 描述
入队 (Insert) add(e) offer(e) 将元素 e 添加到队尾。offer 在队列满时返回 false，add 则抛出异常。
出队 (Remove) remove() poll() 移除并返回队首的元素。poll 在队列为空时返回 null，remove 则抛出异常。
检查队首 (Examine) element() peek() 返回队首的元素，但不移除它。peek 在队列为空时返回 null，element 则抛出异常。

使用建议： 在大多数情况下，推荐使用 offer, poll, peek 这一组方法，因为它们通过返回 null 或 false 来处理边界情况，使得代码逻辑更清晰，无需通过 try-catch 来处理异常。

2. Deque：双端队列

Deque (Double Ended Queue，发音同 “deck”) 接口继承自 Queue 接口，它代表一个双端队列。双端队列允许在队列的两端进行元素的添加和移除操作，因此它既可以作为队列 (FIFO) 使用，也可以作为栈 (Stack, LIFO - Last-In, First-Out) 使用。

Deque 提供了如 addFirst, addLast, pollFirst, pollLast 等更直观的方法。

  • 作队列 (Queue) 使用: addLast (入队), pollFirst (出队)。
  • 作栈 (Stack) 使用: addFirst (入栈), pollFirst (出栈)。

官方推荐使用 Deque 的实现类（如 ArrayDeque）来替代老旧的 Stack 类。

3. Queue/Deque 的主要实现类

a. LinkedList：基于链表的双端队列

我们之前在学习 List 时已经接触过 LinkedList。因为它内部是双向链表结构，所以在头部和尾部添加或删除节点都非常高效，时间复杂度为 O(1)。这使得 LinkedList 天然地成为 Queue 和 Deque 的一个优秀实现。

  • 优点: 增删快，没有容量限制。
  • 缺点: 内存开销相对较大（因为需要存储节点和指针），随机访问性能差（但队列操作不涉及随机访问）。

b. ArrayDeque：基于数组的双端队列

ArrayDeque 是 Deque 接口的一个更现代、更高效的实现。它的底层是一个可动态调整大小的循环数组。

    subgraph "ArrayDeque (Circular Array)"
        direction LR
        A["Index 0"] --- B["Index 1"] --- C["Index 2 (tail)"] --- D["Index 3 (head)"] --- E["Index 4"] --- F["..."]
        C -- "wraps around" --> D
    end
    D --> |pollFirst| Element1
    C --> |addLast| Element2
  • 内部原理: ArrayDeque 内部维护一个数组和两个指针 head 和 tail。当从头部或尾部添加/删除元素时，只需移动指针。当指针到达数组边界时，它会自动“绕”到数组的另一端，形成一个环形结构，这避免了数组元素的物理移动。
  • 优点:
    • 作为队列或栈使用时，其性能通常优于 LinkedList，因为它具有更好的局部性（数组内存连续），对 CPU 缓存更友好。
    • 没有 LinkedList 的指针开销，内存效率更高。
  • 缺点: 不适合包含 null 元素。

ArrayDeque vs LinkedList： 在需要一个队列或栈时，ArrayDeque 通常是比 LinkedList 更好的选择，除非你需要在集合中间进行元素的增删（这是 List 的操作，而非 Queue 的典型操作）。

c. PriorityQueue：优先队列

PriorityQueue 是一种特殊的队列，它不遵循 FIFO 原则，而是根据元素的优先级进行出队。每次 poll() 操作，返回的都是队列中优先级最高的元素。

  • 内部原理: PriorityQueue 的底层是一个二叉堆 (Binary Heap)，通常是最小堆。这意味着队首 peek() 永远是最小的元素（根据自然排序或 Comparator）。
  • 排序机制:
    1. 自然排序: 存入的元素必须实现 Comparable 接口。
    2. 定制排序: 创建 PriorityQueue 时传入一个 Comparator 对象。
  • 性能: offer (入队) 和 poll (出队) 的时间复杂度都是 O(log n)。
  • 使用场景: 非常适合需要动态找出“最大/最小”元素的场景，如任务调度中的最高优先级任务、Top-K 问题等。

4. 总结与选择

  • 需要一个先进先出 (FIFO) 的队列？ -> ArrayDeque 是首选。
  • 需要一个后进先出 (LIFO) 的栈？ -> ArrayDeque 是首选。
  • 需要一个根据优先级自动排序的队列？ -> PriorityQueue。
  • 需要在队列的中间进行增删操作（不常见）？ -> LinkedList。
