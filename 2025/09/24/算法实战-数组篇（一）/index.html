这是我们“算法与设计模式实战”系列的第一篇，主题是数组。数组是最基础、最重要的数据结构之一，我们将通过两道经典的算法题来深入理解它的应用。

题目一：最短路径

1. 题目描述

给定一个表示地图上坐标的 2D 数组 map，地图上只有值 0, 1, 2。

  • 0 表示可以通过的路径。
  • 1 表示不可通过的墙壁。
  • 2 表示目标位置。

从坐标 [0, 0] 开始，每次只能上、下、左、右移动一格。请找出从起点到目标位置的最短路径，并返回路径的长度。如果无法到达，则返回 -1。

2. 解题思路分析

这个问题本质上是在一个无权图 (Unweighted Graph) 中寻找两个节点之间的最短路径。对于这类问题，广度优先搜索 (Breadth-First Search, BFS) 是标准且最高效的解决方案。

BFS 的核心特性是逐层遍历。它从起点开始，先访问所有距离为 1 的节点，然后是所有距离为 2 的节点，以此类推。这种性质保证了当 BFS 第一次找到目标节点时，它所经过的路径必然是最短的。

4. 复杂度分析

  • 时间复杂度: O(R * C)，其中 R 是行数，C 是列数。
  • 空间复杂度: O(R * C)，用于 visited 数组和队列。

题目二：和为零的子矩阵

1. 题目描述

给定一个二维整数矩阵 matrix，找出是否存在一个非空的子矩阵，使得其内部所有元素的和为零。如果存在，返回 true，否则返回 false。

2. 核心思路：降维打击

解决这个问题的关键，是将二维问题降维成我们更熟悉的一维问题：“和为零的子数组”。

  1. 固定边界: 我们用两层循环遍历并固定子矩阵的左、右边界 (leftCol, rightCol)。
  2. 压平成一维: 对于固定的左右边界，我们将这两列之间的所有行进行求和，形成一个一维的“行和”数组 rowSums。
  3. 一维求解: 在 rowSums 这个一维数组上，使用“前缀和 + 哈希表”的方法，在 O(行数) 的时间内判断是否存在和为零的连续子数组。
  4. 重复: 重复以上步骤，直到遍历完所有可能的列组合。

3. Java 源码实现

4. 复杂度分析

  • 时间复杂度: O(列数² * 行数)。
  • 空间复杂度: O(行数)，用于 rowSums 数组和哈希表。
