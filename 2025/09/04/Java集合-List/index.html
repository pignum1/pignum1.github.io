Java 集合框架：List 接口详解

在 Java 的集合框架 (Collections Framework) 中，List 是一个核心接口，它代表一个有序的集合，并且允许包含重复的元素。它是我们日常开发中最常打交道的集合类型之一。

1. List 接口的核心特性

  • 有序性 (Ordered): List 会维护元素的插入顺序。你以什么顺序添加元素，遍历时就会得到什么顺序。
  • 允许重复 (Allows Duplicates): 你可以向 List 中添加多个相同（通过 equals() 方法判断）的元素。
  • 基于索引 (Index-based): List 中的每个元素都有一个唯一的、从 0 开始的整数索引，可以通过索引精确地访问、修改或删除元素。

常用方法

List 接口提供了一系列基于索引的操作方法：

  • void add(int index, E element): 在指定索引处插入元素。
  • E get(int index): 获取指定索引处的元素。
  • E set(int index, E element): 替换指定索引处的元素。
  • E remove(int index): 移除指定索引处的元素。
  • int indexOf(Object o): 返回指定元素首次出现的索引。

2. ArrayList：高效的随机访问

ArrayList 是 List 接口最常用的实现类。

内部原理

ArrayList 的底层是基于一个动态数组 (dynamic array) 实现的。当你创建一个 ArrayList 时，它内部会初始化一个数组来存储元素。

    graph LR
    A["0: Apple"] --> B["1: Banana"] --> C["2: Cherry"] --> D["3: (empty)"] --> E["4: (empty)"]
  • 扩容机制: 当你不断向 ArrayList 添加元素，导致内部数组容量不足时，它会自动进行扩容。通常，它会创建一个新的、更大的数组（一般是原容量的 1.5 倍），然后将旧数组中的所有元素复制到新数组中。这是一个时间开销较大的操作。

性能特点

  • 查询快 (高效随机访问): 由于底层是数组，通过索引 get(index) 来访问元素的时间复杂度是 O(1)，非常快。
  • 中间增删慢: 在数组的中间位置插入或删除元素（add(index, e) 或 remove(index)）需要移动该位置之后的所有元素，时间复杂度是 O(n)，当数据量大时会很慢。但在末尾添加元素通常很快（O(1)），除非触发了扩容。

使用场景

当你需要频繁地根据索引读取数据，而增删操作相对较少（尤其是在列表的中间位置）时，ArrayList 是最佳选择。

3. LinkedList：高效的插入与删除

LinkedList 是 List 接口的另一个重要实现类。

内部原理

LinkedList 的底层是基于一个双向链表 (doubly-linked list) 实现的。它不使用连续的内存空间。每个元素都是一个“节点” (Node)，每个节点除了存储数据外，还存储了指向前一个节点和后一个节点的引用。

性能特点

  • 增删快: 在任意位置插入或删除元素时，只需要修改目标位置前后节点的引用即可，时间复杂度是 O(1)（前提是已经定位到该节点）。
  • 查询慢: 由于不是连续存储，LinkedList 不支持高效的随机访问。当你需要 get(index) 一个元素时，它必须从链表的头部或尾部开始，一个一个地遍历直到找到目标索引，时间复杂度是 O(n)。

使用场景

当你需要进行大量的插入和删除操作，而对随机访问的需求不高时，LinkedList 是更优的选择。

4. ArrayList vs. LinkedList 总结

特性/操作 ArrayList LinkedList
底层结构 动态数组 双向链表
随机访问 (get) 极快 (O(1)) 慢 (O(n))
末尾添加 (add) 快 (O(1)) 快 (O(1))
中间添加/删除 慢 (O(n)) 快 (O(1))
内存占用 较少，连续 较多 (需要额外空间存引用)

简单选择原则：

  • 读多写少，用 ArrayList。
  • 写多读少，用 LinkedList。
  • 在不确定或绝大多数情况下，优先使用 ArrayList，因为它在实际应用中的综合性能通常更好。

5. 常见面试题

Q1: ArrayList 是如何扩容的？

A: 当 ArrayList 的元素数量超过其内部数组的容量时，它会进行扩容。通常，它会创建一个容量为原容量1.5倍的新数组，然后将旧数组的所有元素复制到新数组中。这是一个 O(n) 的操作，因此频繁的扩容会影响性能。

Q2: ArrayList 和 LinkedList 是线程安全的吗？

A: 都不是线程安全的。如果在多线程环境下对它们进行修改，可能会导致 ConcurrentModificationException 或数据不一致。如果需要线程安全，可以使用 Collections.synchronizedList() 方法包装现有的 List，或者使用 java.util.concurrent 包下的 CopyOnWriteArrayList。

Q3: 如何在遍历 List 的同时安全地删除元素？

A: 错误的方式是直接使用 for-each 循环并调用 list.remove()，这会抛出 ConcurrentModificationException。正确的方式是使用迭代器 (Iterator) 的 remove() 方法。

Q4: 如何在单次遍历中找到 LinkedList 的中间元素？

A: 使用“快慢指针”法。定义两个指针，一个“慢指针”每次移动一步，一个“快指针”每次移动两步。当快指针到达链表末尾时，慢指针正好位于中间位置。这是一个经典的链表算法题。表末尾时，慢指针正好位于中间位置。这是一个经典的链表算法题。
