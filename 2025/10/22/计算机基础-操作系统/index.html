计算机操作系统核心知识

操作系统（Operating System, OS）是管理计算机硬件与软件资源的系统软件，同时也是计算机系统的内核与基石。它为用户和其他软件提供了与硬件交互的接口。理解操作系统的核心概念，对于编写高性能、高可靠性的应用程序至关重要。

1. 进程与线程 (Process & Thread)

这是操作系统中最核心、最基本的概念，也是理解并发编程的基础。

进程 (Process)

  • 定义: 进程是程序的一次执行过程，是系统进行资源分配和调度的基本单位。每个进程都有自己独立的内存空间。
  • 组成: 主要由程序代码、数据、进程控制块 (PCB) 组成。PCB 是进程存在的唯一标识。
  • 特性:
    • 动态性: 进程是程序的一次执行，有生命周期。
    • 独立性: 作为资源分配的基本单位，各进程的地址空间相互独立。
    • 并发性: 多个进程可以在单个处理器上并发执行。
  • 状态: 创建 (New)、就绪 (Ready)、运行 (Running)、阻塞 (Blocked)、终止 (Terminated)。

线程 (Thread)

  • 定义: 线程是进程的一个执行实体，是 CPU 调度和分派的基本单位。它是比进程更小的能独立运行的基本单位。
  • 关系: 一个进程可以包含多个线程，这些线程共享该进程的内存空间和资源（如代码段、数据段、打开的文件）。但每个线程有自己独立的程序计数器、栈和局部变量。
  • 优势:
    • 开销小: 创建和切换线程的开销远小于进程。
    • 并发度高: 在多核 CPU 环境下，多线程可以真正地并行执行。
    • 资源共享方便: 线程间共享数据无需复杂的进程间通信 (IPC)。

进程与线程的区别

特性 进程 线程
资源分配 资源分配的基本单位 不拥有资源，但可访问隶属进程的资源
调度 独立调度的基本单位 CPU 调度的基本单位
系统开销 创建、销毁、切换开销大 创建、销毁、切换开销小
内存空间 拥有独立的内存空间 共享进程的内存空间
通信 需通过 IPC (管道, 消息队列, 共享内存) 可直接读写共享变量，但需注意同步问题

2. 内存管理 (Memory Management)

内存管理的主要职责是有效地分配和管理内存资源，确保进程可以安全、高效地使用内存。

内存分配方式

  • 连续分配: 为用户进程分配一块连续的内存空间。包括单一连续分配、固定分区分配、动态分区分配。
  • 非连续分配: 允许将一个进程分散地装入许多不相邻的内存分区。
    • 分页 (Paging): 将内存空间划分为大小相等的“页框”，将进程的逻辑地址空间也划分为大小相等的“页”。通过页表来记录逻辑页到物理页框的映射关系。
    • 分段 (Segmentation): 按程序的逻辑结构（如代码段、数据段、堆栈段）来划分内存，段的长度可变。通过段表来记录段号到物理地址的映射。
    • 段页式 (Segmentation with Paging): 先分段，再将每个段分页。

虚拟内存 (Virtual Memory)

  • 定义: 虚拟内存是一种内存管理技术，它为每个进程提供了一个大的、连续的、私有的地址空间，使其看起来拥有比物理内存更多的内存。它通过将部分暂时不用的内存数据存放到磁盘上，在需要时再调入内存来实现。
  • 核心技术: 请求分页、页面置换算法。
  • 页面置换算法: 当发生缺页中断且没有空闲页框时，需要选择一个页面进行淘汰。
    • OPT (最佳置换算法): 理论最优，但无法实现。
    • FIFO (先进先出): 实现简单，但可能产生“贝拉迪异常”。
    • LRU (最近最少使用): 性能好，但实现开销大。通常使用近似算法，如时钟算法 (Clock)。
    • LFU (最不经常使用): 考虑了使用频率，但实现复杂。

3. 文件系统 (File System)

文件系统是操作系统中负责管理和组织文件的方法。

  • 文件的逻辑结构: 流式文件、记录式文件。
  • 目录结构: 单级目录、两级目录、树形目录。
  • 文件分配方式: 连续分配、链接分配（隐式/显式）、索引分配。
  • 磁盘空间管理: 空闲区表、空闲链表、位示图。

4. I/O (Input/Output)

I/O 管理是操作系统的重要组成部分，负责管理 I/O 设备和完成 I/O 操作。

I/O 模型

  • 阻塞 I/O (Blocking I/O): 发起 I/O 请求后，进程阻塞直到数据准备好并从内核复制到用户空间。
  • 非阻塞 I/O (Non-blocking I/O): 发起 I/O 请求后，立即返回。进程需要不断轮询内核，检查数据是否准备好。
  • I/O 多路复用 (I/O Multiplexing): 通过 select, poll, epoll 等机制，单个进程可以同时监视多个文件描述符（Socket），一旦某个描述符就绪，就通知应用程序进行读写操作。这是实现高并发网络编程的关键（如 Netty, Redis）。
  • 信号驱动 I/O (Signal-driven I/O): 进程发起请求后立即返回，当数据准备好时，内核向进程发送一个信号，进程在信号处理函数中进行 I/O 操作。
  • 异步 I/O (Asynchronous I/O): 进程发起 I/O 请求后立即返回，内核在完成所有操作（包括将数据从内核复制到用户空间）后，再通知进程。

零拷贝 (Zero-copy)

  • 定义: 零拷贝是一种避免 CPU 将数据从一块内存复制到另一块内存的技术，从而减少上下文切换和 CPU 拷贝次数，以提高性能。
  • 常见实现: mmap (内存映射)、sendfile (Linux 系统调用)。Kafka、Netty 等高性能框架都广泛使用了零拷贝技术。
