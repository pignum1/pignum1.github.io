高速缓冲存储器 (Cache)

高速缓冲存储器（Cache）是位于 CPU 和主存之间的一种容量小、速度快的高速存储器。它的存在是为了缓解 CPU 与主存之间巨大的速度差异。Cache 的设计和性能是现代计算机体系结构中的核心问题之一。

1. 工作原理：局部性原理

Cache 的有效性完全建立在局部性原理之上：

  • 时间局部性: 最近被访问的数据很可能在不久的将来再次被访问。
  • 空间局部性: 一个数据被访问时，其地址相邻的数据也很可能即将被访问。

基于此，当 CPU 访问内存时，它会先在 Cache 中查找。如果找到（缓存命中, Cache Hit），则直接从 Cache 中获取数据，速度极快。如果未找到（缓存未命中, Cache Miss），则需要从主存中读取数据，同时会将该数据及其相邻数据组成一个数据块 (Cache Line) 加载到 Cache 中，以备将来使用。

2. 映射方式 (Mapping)

映射方式规定了主存中的一个数据块可以被放置到 Cache 中的哪个位置。主要有三种映射方式：

2.1 直接映射 (Direct Mapped)

  • 规则: 主存中的每一个块只能映射到 Cache 中的一个固定位置。
  • 地址结构: 主存地址被分为三部分：标记 (Tag) | 索引 (Index) | 块内地址 (Offset)。
    • Index 用于确定该数据块在 Cache 中的位置（行号）。
    • Tag 用于区分映射到同一 Cache 行的不同主存块。
  • 优点: 实现简单，硬件成本低，地址转换速度快。
  • 缺点: 冲突率高。如果两个频繁访问的数据块恰好被映射到同一个 Cache 行，即使 Cache 中还有其他空闲位置，它们也会不断地相互替换，导致缓存命中率急剧下降。这种情况被称为抖动 (Thrashing)。

2.2 全相联映射 (Fully Associative)

  • 规则: 主存中的任何一个块可以映射到 Cache 中的任何位置。
  • 地址结构: 主存地址被分为两部分：标记 (Tag) | 块内地址 (Offset)。
  • 查找过程: 需要将主存地址的 Tag 与 Cache 中所有行的 Tag 并行比较，以确定数据是否存在。
  • 优点: 冲突率最低，Cache 空间利用率最高。
  • 缺点: 比较电路非常复杂，硬件成本极高，查找速度慢。只适用于非常小的 Cache。

2.3 组相联映射 (Set-Associative)

组相联映射是直接映射和全相联映射的一种折衷方案，也是现代 CPU Cache 最常用的方式。

  • 规则: 将 Cache 分为若干个组 (Set)。主存中的每一个块只能映射到 Cache 中的一个固定组，但在该组内的任何位置都是自由的。
  • 地址结构: 主存地址被分为三部分：标记 (Tag) | 组索引 (Set Index) | 块内地址 (Offset)。
    • Set Index 用于确定该数据块可以存放在哪个组。
    • Tag 用于在该组内进行区分。
  • N路组相联: 如果一个组内有 N 个 Cache 行，就称为 N 路组相联。例如，2 路组相联意味着一个主存块可以被放置在一个特定组的两个位置中的任意一个。
  • 优点: 在冲突率和硬件成本之间取得了很好的平衡。
  • 缺点: 实现比直接映射复杂，但比全相联映射简单得多。

3. 替换算法 (Replacement Algorithms)

当发生缓存未命中，并且需要加载的数据块对应的 Cache 行（或组）已满时，就需要选择一个已有的数据块进行替换。替换算法的目标是，尽可能地替换掉未来最不可能被访问的数据块。

  • LRU (Least Recently Used - 最近最少使用): 替换掉最长时间未被访问过的数据块。这是性能最好、最符合局部性原理的算法之一，但硬件实现非常复杂，需要为每个 Cache 行维护一个访问时间戳。
  • FIFO (First-In, First-Out - 先进先出): 替换掉最早进入 Cache 的数据块。实现简单，但性能不佳，因为它可能会替换掉一个经常被访问的“老”数据。
  • Random (随机替换): 随机选择一个数据块进行替换。实现最简单，但性能不稳定。

在实际的硬件中，通常使用 LRU 的近似算法，如伪 LRU (PLRU)。

4. 写策略 (Write Policies)

写策略处理的是当 CPU 修改了 Cache 中的数据后，如何以及何时将这些修改同步回主存的问题。

  • 写回法 (Write-Back)

    • 策略: CPU 的写操作只针对 Cache。仅当该 Cache 行被替换出去时，才将其内容写回主存。为了标识该行是否被修改过，每个 Cache 行都有一个**“脏位” (Dirty Bit)**。
    • 优点: 写入速度快，减少了对主存的访问次数，节省了内存带宽。
    • 缺点: 实现较复杂，存在数据不一致的风险（在写回之前，主存中的数据是旧的）。

  • 全写法 (Write-Through)

    • 策略: CPU 的写操作会同时写入 Cache 和主存。
    • 优点: 实现简单，能时刻保证 Cache 和主存的数据一致性。
    • 缺点: 写入速度慢，因为每次写操作都需要访问慢速的主存，可能会造成 CPU 阻塞。

为了缓解全写法中 CPU 的等待问题，通常会引入一个写缓冲 (Write Buffer)。CPU 将数据写入 Cache 和写缓冲后就可以继续执行，由写缓冲负责在后台将数据慢慢写入主存。
