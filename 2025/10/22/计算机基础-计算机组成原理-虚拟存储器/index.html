虚拟存储器 (Virtual Memory)

虚拟存储器是现代计算机系统内存管理的一项核心技术。它为每个进程提供了一个独立的、连续的、巨大的逻辑地址空间，而无需关心物理内存的实际大小和碎片化情况。这使得程序员可以编写和运行远大于实际物理内存的程序。

虚拟存储器的实现主要依赖于主存-辅存这一层次的数据交换，并由硬件和操作系统共同完成。

1. 基本概念

  • 逻辑地址 (Logical Address) / 虚拟地址 (Virtual Address): CPU 生成的、程序员视角下的地址。它位于虚拟地址空间中。
  • 物理地址 (Physical Address): 物理内存中真实的地址。
  • 内存管理单元 (Memory Management Unit, MMU): CPU 内部的一个硬件单元，负责将逻辑地址实时地转换为物理地址。

2. 虚拟存储器的实现方式

主要有三种方式：页式、段式和段页式。其中，页式虚拟存储是目前最主流的方式。

2.1 页式存储管理 (Paging)

  • 基本思想:

    1. 将进程的逻辑地址空间划分为大小相等的页 (Page)。
    2. 将物理内存划分为与页大小相等的页框 (Page Frame)。
    3. 在程序加载时，可以按需将任意页加载到任意空闲的页框中，无需连续。
    4. 操作系统为每个进程维护一个页表 (Page Table)，用于记录逻辑页到物理页框的映射关系。

  • 地址转换过程:

    1. CPU 产生一个逻辑地址，该地址分为两部分：逻辑页号 (Page Number) 和页内偏移 (Offset)。
    2. MMU 根据逻辑页号去查询当前进程的页表。
    3. 从页表中找到对应的物理页框号。
    4. 将物理页框号与页内偏移拼接，形成最终的物理地址。

    物理地址 = 物理页框号 + 页内偏移

  • 缺页中断 (Page Fault): 如果在查询页表时，发现对应的页不在内存中（页表项中的有效位为 0），MMU 会产生一个缺页中断。操作系统接管后，会执行以下操作：

    1. 在辅存（硬盘）中找到该页。
    2. 在物理内存中找到一个空闲的页框。如果没有空闲页框，则根据页面替换算法选择一个页面进行淘汰。
    3. 将所需的页从辅存加载到该页框中。
    4. 更新页表，建立新的映射关系。
    5. 返回到原来的指令，重新执行。

2.2 快表 (Translation Lookaside Buffer, TLB)

由于页表本身存放在主存中，每次地址转换都至少需要访问一次内存来查询页表，这会使访存速度降低一半。为了解决这个问题，引入了快表 (TLB)。

  • 定义: TLB 是一个位于 CPU 内部的、容量小但速度极快的高速缓存，专门用于缓存最近使用过的页表项。
  • 工作流程:
    1. CPU 产生逻辑地址后，MMU 首先用逻辑页号在 TLB 中查找。
    2. TLB 命中 (TLB Hit): 如果在 TLB 中找到对应的页表项，直接获取物理页框号，完成地址转换。这个过程极快。
    3. TLB 未命中 (TLB Miss): 如果在 TLB 中未找到，则需要去访问主存中的页表。找到页表项后，一方面用它来完成地址转换，另一方面将其缓存到 TLB 中，以备将来使用。如果 TLB 已满，则需要根据替换算法（如 LRU）替换掉一个条目。

通过使用 TLB，绝大多数的地址转换都可以在 CPU 内部高速完成，大大提高了性能。

2.3 段式与段页式存储

  • 段式存储 (Segmentation): 按程序的逻辑结构（如代码段、数据段、堆栈段）来划分地址空间，段的长度可变。优点是便于共享和保护，但容易产生外部碎片。
  • 段页式存储 (Segmentation with Paging): 先将程序分段，再将每个段分页。它结合了段式和页式的优点，是现代操作系统（如 Linux）实际采用的方式。

3. 图形化对比：页式、段式、段页式

为了更直观地理解这三者的区别，我们通过图表来对比它们的地址结构和转换过程。

页式 (Paging)

  • 核心思想: 内存被划分为大小固定的页，逻辑地址是一维的。
  • 优点: 内存利用率高，没有外部碎片。
  • 缺点: 页面不是一个逻辑整体，不便于共享和保护。

段式 (Segmentation)

  • 核心思想: 内存按逻辑功能划分为大小可变的段，逻辑地址是二维的（段号 + 段内偏移）。
  • 优点: 便于按逻辑模块进行共享和保护。
  • 缺点: 会产生外部碎片，导致内存利用率降低。

段页式 (Segmented Paging)

  • 核心思想: 先分段，再分页。结合了两者的优点。
  • 地址结构: 逻辑地址包含段号、段内页号、页内偏移。
  • 地址转换: 需要先查段表，再查页表。

总结

虚拟存储器通过离散分配（分页/分段）和按需调入（缺页中断）两大技术，为用户提供了一个远大于实际物理内存的编程环境，并带来了以下好处：

  • 简化了内存管理: 程序员无需关心物理内存的实际大小和地址，只需在连续的逻辑地址空间上编程。
  • 提高了内存利用率: 多个进程可以共享物理内存，且只有程序的一部分需要被加载到内存中即可运行。
  • 提供了内存保护: 每个进程拥有独立的地址空间，一个进程无法直接访问另一个进程的内存，保证了系统的安全性。
