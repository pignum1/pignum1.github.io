CPU 的功能和基本结构

中央处理器（Central Processing Unit, CPU）是计算机的运算和控制核心。它的主要功能是解释计算机指令以及处理计算机软件中的数据。CPU 的工作可以概括为：取指令、分析指令、执行指令。

1. CPU 的两大核心功能

  1. 指令控制: 控制程序的执行顺序。CPU 通过程序计数器（PC）来确定下一条指令的地址，实现程序的顺序执行、分支、循环和跳转。
  2. 操作控制: 一条指令可以分解为多个微操作。CPU 的控制器负责产生每个微操作所需的控制命令，并将它们发送到相应的部件，以控制这些部件执行相应的操作。

2. CPU 的基本结构

CPU 主要由运算器 (ALU) 和控制器 (Control Unit) 两大部分组成。

2.1 运算器 (Arithmetic Logic Unit, ALU)

运算器是 CPU 中负责执行数据处理的部分，即执行算术运算和逻辑运算。

  • 功能:
    • 算术运算: 加、减、乘、除等。
    • 逻辑运算: 与、或、非、异或等。
  • 核心组成:
    1. 算术逻辑单元 (ALU): 是运算器的核心，负责执行具体的算术和逻辑操作。
    2. 累加寄存器 (Accumulator, ACC): 一个通用的寄存器，用于暂存 ALU 的操作数和运算结果。
    3. 数据缓冲寄存器 (Data Register, DR): 用于暂存从内存读取或写入内存的数据。
    4. 状态条件寄存器 (Program Status Word, PSW): 用于存放运算过程中的状态信息和控制信息，如进位标志 (Carry)、溢出标志 (Overflow)、零标志 (Zero)、负标志 (Negative) 等。这些标志位常被用于条件转移指令的判断。

2.2 控制器 (Control Unit, CU)

控制器是整个 CPU 的指挥中枢。它负责协调计算机各个部分的工作，确保指令能够被正确地执行。

  • 功能: 从主存中取出指令，进行译码（分析指令要做什么），然后根据指令的要求，产生一系列的控制信号，发送给运算器、主存和 I/O 设备，以控制它们完成相应的操作。
  • 核心组成:
    1. 程序计数器 (Program Counter, PC): 也叫指令指针。它存放着下一条将要执行指令的内存地址。当一条指令被取出后，PC 会自动指向下一条指令。
    2. 指令寄存器 (Instruction Register, IR): 用于存放当前正在执行的指令。当 CPU 从内存中取出一条指令后，会先将其放入 IR 中。
    3. 指令译码器 (Instruction Decoder, ID): 对指令寄存器（IR）中的操作码部分进行分析和解释，确定指令要执行的操作类型。
    4. 时序控制电路: 产生各种时序信号，以确保 CPU 的各个操作能够按正确的顺序进行。

3. 寄存器组 (Registers)

寄存器是 CPU 内部的、容量小但速度极快的高速存储部件。它们用于暂时存放程序执行过程中的指令、数据和地址。

  • 通用寄存器: 可用于多种目的，如存放操作数、地址等。程序员可以在汇编语言中直接访问它们。
  • 专用寄存器: 有特定用途的寄存器，主要就是构成上述运算器和控制器的核心部件，例如：
    • PC (程序计数器)
    • IR (指令寄存器)
    • MAR (Memory Address Register): 内存地址寄存器，用于存放 CPU 将要访问的内存单元的地址。
    • MDR (Memory Data Register): 内存数据寄存器，用于暂存从内存读出或要写入内存的数据。
    • PSW (程序状态字寄存器)

4. CPU 的工作流程（指令周期）

CPU 执行一条指令的完整过程被称为一个指令周期，它通常包含以下几个阶段：

  1. 取指周期 (Fetch Cycle):

    • CPU 根据 PC 中的地址，从主存中取出指令。
    • 将取出的指令存入指令寄存器 IR。
    • PC 自动加 1，指向下一条指令。

  2. 译码周期 (Decode Cycle):

    • 控制器中的指令译码器（ID）对 IR 中的指令进行分析，确定操作类型和操作数地址。

  3. 执行周期 (Execute Cycle):

    • 控制器根据指令的功能，发出相应的控制信号。
    • 运算器（ALU）根据控制信号，对操作数执行规定的算术或逻辑运算。

这个“取指-译码-执行”的循环不断重复，构成了计算机程序执行的基本模型。
