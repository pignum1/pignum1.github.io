JavaSE - 常用 API

Java Development Kit (JDK) 提供了大量功能强大的 API (Application Programming Interface)，这些 API 是我们日常开发的基础工具。熟练掌握常用 API 能极大地提高开发效率和代码质量。

1. String 类

String 是 Java 中最重要、最常用的类之一，用于表示字符串。String 对象是不可变的 (immutable)，一旦创建，其内容就不能被修改。

1.1 String 的不可变性

  • 优点:
    1. 线程安全: 因为不可变，所以可以在多线程环境中安全共享，无需同步。
    2. 可作为 HashMap 的键: String 的 hashCode() 是缓存的，因为字符串内容不变，所以哈希码也不变，提高了 HashMap 的性能。
    3. 字符串常量池 (String Pool): 不可变性是字符串常量池实现的基础，可以节省大量内存。
  • 缺点: 每次对 String 对象进行修改（如拼接、替换），都会创建一个新的 String 对象，这在大量操作时会产生性能开销和内存垃圾。

1.2 常用方法

  • int length(): 返回字符串长度。
  • char charAt(int index): 返回指定索引处的字符。
  • boolean equals(Object anObject): 比较字符串内容是否相同（区分大小写）。
  • boolean equalsIgnoreCase(String anotherString): 比较字符串内容是否相同（不区分大小写）。
  • String substring(int beginIndex, int endIndex): 截取子字符串。
  • String[] split(String regex): 根据正则表达式分割字符串。
  • boolean contains(CharSequence s): 判断是否包含指定字符序列。
  • int indexOf(String str): 返回子字符串首次出现的索引，不存在则返回 -1。
  • String trim(): 去除字符串首尾的空白字符。
  • String toLowerCase() / toUpperCase(): 转换为小写/大写。
  • boolean isEmpty(): 判断字符串是否为空（长度为0）。

2. StringBuilder 和 StringBuffer

当需要对字符串进行频繁修改时，应使用 StringBuilder 或 StringBuffer，它们是可变的字符序列。

  • StringBuilder (推荐使用)

    • 线程不安全，但性能更高。
    • 适用于单线程环境下对字符串进行大量修改的场景。
    • 常用方法: append(), insert(), delete(), reverse()。

  • StringBuffer

    • 线程安全，其方法大多用 synchronized 修饰。
    • 性能较低，适用于多线程环境下共享字符串缓冲区的场景。

3. Math 类

Math 类提供了用于执行基本数学运算的静态方法，如初等指数、对数、平方根和三角函数。

  • Math.abs(a): 返回绝对值。
  • Math.max(a, b) / Math.min(a, b): 返回最大/最小值。
  • Math.pow(a, b): 返回 a 的 b 次幂。
  • Math.sqrt(a): 返回 a 的平方根。
  • Math.random(): 返回一个 [0.0, 1.0) 之间的随机 double 值。
  • Math.round(a): 四舍五入。
  • Math.ceil(a): 向上取整。
  • Math.floor(a): 向下取整。

4. 包装类 (Wrapper Classes)

Java 为每种基本数据类型都提供了一个对应的包装类，使得基本数据类型可以像对象一样被操作。

基本类型 包装类
byte Byte
short Short
int Integer
long Long
float Float
double Double
char Character
boolean Boolean
  • 自动装箱 (Autoboxing): 自动将基本数据类型转换为包装类对象。Integer i = 100;
  • 自动拆箱 (Unboxing): 自动将包装类对象转换为基本数据类型。int n = i;
  • 常用方法: Integer.parseInt(String s) (将字符串转换为 int), String.valueOf(int i) (将 int 转换为字符串)。

5. 日期时间 API

5.1 旧版 API (Date, Calendar, SimpleDateFormat)

  • 缺点: 设计混乱、可变性导致线程不安全、时区处理复杂、月份从 0 开始等问题。
  • 不推荐在新代码中使用。

5.2 Java 8 新日期时间 API (java.time 包)

Java 8 引入了一套全新的、设计优良的日期时间 API，解决了旧 API 的所有问题。它是不可变的和线程安全的。

  • 核心类:

    • LocalDate: 表示日期（年-月-日）。
    • LocalTime: 表示时间（时:分:秒.纳秒）。
    • LocalDateTime: 表示日期和时间，是 LocalDate 和 LocalTime 的结合。
    • Instant: 表示时间戳，精确到纳秒，用于机器时间。
    • Duration: 表示两个时间之间的间隔（以秒和纳秒为单位）。
    • Period: 表示两个日期之间的间隔（以年、月、日为单位）。
    • DateTimeFormatter: 用于格式化和解析日期时间对象。
