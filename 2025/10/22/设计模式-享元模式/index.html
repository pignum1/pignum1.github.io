什么是享元模式？

享元模式 (Flyweight Pattern) 是一种结构型设计模式，它旨在通过共享尽可能多的相似对象来最小化内存使用或计算开销。当程序需要创建大量相似的对象，而这些对象的大部分状态都可以被外部化和共享时，此模式尤其有效。

享元模式的核心在于区分对象的内部状态 (Intrinsic State) 和外部状态 (Extrinsic State)。

  • 内部状态: 存储在享元对象内部，并且不随环境改变，可以被多个对象共享。例如，一个字符的字形、颜色。
  • 外部状态: 由客户端计算或存储，在使用享元对象时才传递给它，并且会随环境改变。例如，一个字符在文档中的位置坐标。

享元模式通过只存储内部状态，并将外部状态作为方法参数传入，从而实现对象的共享。

模式结构

享元模式主要包含以下角色：

  1. Flyweight (享元接口): 定义了享元对象的接口，通过这个接口，享元可以接收并作用于外部状态。
  2. ConcreteFlyweight (具体享元): 实现了享元接口，并为内部状态增加存储空间。该类的对象必须是可共享的。
  3. UnsharedConcreteFlyweight (非共享具体享元): 并非所有 Flyweight 子类都需要被共享。该类对象通常拥有独特的、不可共享的状态。
  4. FlyweightFactory (享元工厂): 负责创建和管理享元对象。它确保享元被正确地共享。当客户端请求一个享元时，工厂会检查是否已存在一个匹配的实例，如果存在则返回它，否则就创建一个新的实例并存储起来以便将来复用。
  5. Client (客户端): 维护对享元对象的引用，并计算或存储外部状态。

优缺点

优点

  1. 大幅节省内存: 如果相似对象的数量巨大，享元模式可以极大地减少内存消耗。
  2. 提高性能: 减少了对象的创建和销毁次数，从而减轻了垃圾回收的压力，提高了系统性能。

缺点

  1. 增加了系统复杂性: 需要将对象的状态分解为内部和外部状态，这使得系统逻辑更复杂，也要求客户端必须自己管理外部状态。
  2. 线程安全问题: 享元对象是共享的，因此必须确保其内部状态是线程安全的。

经典应用

  1. Java String 池: String 类是 Java 中享元模式最典型的应用。字符串常量池存储了共享的字符串对象。当你创建一个字符串字面量时（如 String s = "abc";），JVM 会先在池中查找是否存在值为 “abc” 的字符串，如果存在则直接返回其引用，否则才创建一个新的字符串对象并放入池中。
  2. Java Integer.valueOf(): Integer.valueOf(int i) 方法对-128到127之间的整数进行了缓存。当你调用 Integer.valueOf(100) 时，它总是返回同一个 Integer 对象，而不是创建一个新的。这也是一种享元模式的应用。
  3. 数据库连接池: 连接池中的数据库连接对象可以被看作是享元对象。连接的配置信息（URL, 用户名, 密码）是内部状态，而正在执行的 SQL 语句和事务是外部状态。客户端从池中获取一个连接，使用它，然后将其归还，实现了连接对象的复用。
