JavaSE - 面向对象编程 (OOP)

面向对象编程（Object-Oriented Programming, OOP）是 Java 语言的核心思想。它是一种程序设计范式，将程序组织为“对象”的集合，每个对象都包含数据（属性）和操作数据的方法（行为）。OOP 的核心优势在于其模块化、可重用性和易于维护性。

1. 类与对象 (Class and Object)

  • 类 (Class): 类是对象的模板或蓝图，它定义了对象的属性（数据）和行为（方法）。类是抽象的，不占用内存空间。

  • 对象 (Object): 对象是类的实例。它是具体的，占用内存空间。通过 new 关键字创建对象。

2. 面向对象三大特性

2.1 封装 (Encapsulation)

  • 定义: 将对象的属性和行为（方法）组合在一起，形成一个独立的单元（类），并对外部隐藏对象的内部实现细节，只暴露有限的接口供外部访问。

  • 实现:

    1. 使用 private 关键字修饰属性，使其不能直接被外部访问。
    2. 提供 public 的 getter（获取属性值）和 setter（设置属性值）方法来间接访问属性。

  • 优点:

    • 提高安全性: 保护数据不被随意修改。
    • 提高代码的独立性和可维护性: 内部实现可以改变，只要接口不变，外部调用不受影响。
    • 降低耦合度: 模块之间通过接口通信，减少了相互依赖。

2.2 继承 (Inheritance)

  • 定义: 允许一个类（子类/派生类）继承另一个类（父类/基类）的属性和方法，从而实现代码的重用。

  • 实现: 使用 extends 关键字。

    • Java 只支持单继承（一个子类只能有一个直接父类），但可以多层继承。

  • 特点:

    • 子类继承父类的所有非 private 属性和方法。
    • 子类可以添加自己的新属性和方法。
    • 子类可以重写 (Override) 父类的方法（方法名、参数列表、返回值类型必须相同）。

  • super 关键字: 用于访问父类的成员（属性、方法、构造器）。

  • final 关键字:

    • 修饰类：表示该类不能被继承。
    • 修饰方法：表示该方法不能被重写。
    • 修饰变量：表示该变量是常量，只能赋值一次。

2.3 多态 (Polymorphism)

  • 定义: 同一个行为在不同的对象上，表现出不同的形态。多态的实现前提是继承和方法重写。

  • 实现:

    1. 向上转型 (Upcasting): 父类引用指向子类对象。父类类型 变量名 = new 子类对象();
    2. 方法重写 (Override): 子类重写父类的方法。

  • 优点:

    • 提高代码的扩展性: 当增加新的子类时，无需修改原有代码。
    • 提高代码的灵活性: 可以使用统一的接口处理不同类型的对象。

3. 抽象类与接口 (Abstract Class and Interface)

3.1 抽象类 (Abstract Class)

  • 定义: 使用 abstract 关键字修饰的类。抽象类不能被实例化，只能被继承。

  • 特点:

    • 可以包含抽象方法（只有方法签名，没有方法体，也用 abstract 修饰）和普通方法。
    • 子类继承抽象类后，必须实现（重写）所有的抽象方法，除非子类也是抽象类。

  • 用途: 用于定义类的通用行为，但某些行为的具体实现留给子类。

3.2 接口 (Interface)

  • 定义: 使用 interface 关键字定义。接口是一种完全抽象的类型，它定义了一组规范，但不提供实现。

  • 特点:

    • 在 Java 8 之前，接口中只能包含 public static final 常量和 public abstract 方法。
    • Java 8 引入了 default 方法和 static 方法。
    • Java 9 引入了 private 方法。
    • 类通过 implements 关键字实现接口，可以实现多个接口（弥补了单继承的不足）。

  • 用途:

    • 定义行为规范。
    • 实现多态。
    • 实现多重继承（行为）。

4. 内部类 (Inner Class)

内部类是定义在另一个类内部的类。它可以访问外部类的所有成员，包括私有成员。

  • 成员内部类: 最普通的内部类，作为外部类的一个成员。
  • 局部内部类: 定义在方法或代码块内部。
  • 匿名内部类: 没有名字的内部类，通常用于实现接口或继承类，且只使用一次。
  • 静态内部类: 用 static 修饰的内部类，不能访问外部类的非静态成员。

5. 包 (Package) 与导入 (Import)

  • 包: 用于组织类，防止命名冲突，并提供访问控制。使用 package 关键字声明。
  • 导入: 使用 import 关键字导入其他包中的类。import com.example.MyClass; 或 import com.example.*;

6. 访问修饰符 (Access Modifiers)

用于控制类、属性、方法的访问权限。
