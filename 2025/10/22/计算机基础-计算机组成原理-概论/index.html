计算机系统概论

计算机组成原理研究的是计算机硬件系统的基本组成部分以及它们之间如何协同工作。理解这些底层概念是编写高效代码、进行系统性能分析和优化的基础。

1. 冯·诺依曼体系结构 (Von Neumann Architecture)

冯·诺依曼体系结构是现代计算机的基础。它由美籍匈牙利数学家冯·诺依曼于1945年提出，其核心思想是**“存储程序” (Stored-Program Computer)**。

五大核心组成部分

  1. 运算器 (Arithmetic Logic Unit, ALU): 负责执行算术运算（加、减、乘、除）和逻辑运算（与、或、非、异或）。
  2. 控制器 (Control Unit, CU): 计算机的指挥中心。它负责从内存中取出指令、分析指令并发出控制信号，协调其他所有组件的工作。
  3. 存储器 (Memory): 用于存放程序（指令）和数据。冯·诺依曼结构的关键在于，指令和数据以同等地位存放在同一存储器中，并可按地址寻访。
  4. 输入设备 (Input Device): 将外部世界的信息（如键盘敲击、鼠标移动）转换成计算机能识别的二进制代码。例如：键盘、鼠标、扫描仪。
  5. 输出设备 (Output Device): 将计算机处理的结果转换成人类可识别的形式。例如：显示器、打印机。

运算器和控制器通常被集成在一起，合称为中央处理器 (Central Processing Unit, CPU)。

核心思想

  • 指令和数据以二进制形式存储: 计算机中所有的指令和数据都用二进制代码表示。
  • 指令和数据存储在同一内存中: 这是与哈佛结构最主要的区别。CPU 可以通过地址直接访问指令和数据。
  • 计算机由五大部件组成: 运算器、控制器、存储器、输入设备、输出设备。
  • 程序驱动: 计算机的工作流程由程序（即一系列指令）来控制。

2. 计算机的基本组成

现代计算机通常被划分为三个主要部分：CPU、主存和I/O设备。

CPU (中央处理器)

  • 功能: 指令控制、操作控制、时间控制、数据加工。
  • 组成:
    • 运算器 (ALU): 执行算术和逻辑运算。
    • 控制器 (CU): 解释指令，发出控制信号。
    • 寄存器组 (Registers): CPU 内部的高速存储单元，用于暂存指令、数据和地址。常见的寄存器有：
      • PC (Program Counter): 程序计数器，存放下一条要执行指令的地址。
      • IR (Instruction Register): 指令寄存器，存放当前正在执行的指令。
      • MAR (Memory Address Register): 内存地址寄存器，存放将要访问的内存单元的地址。
      • MDR (Memory Data Register): 内存数据寄存器，存放从内存读取或写入内存的数据。

主存储器 (Main Memory)

  • 也称为内存，用于存放正在运行的程序和数据。
  • 特点是速度快，但容量相对较小，且断电后数据会丢失（易失性）。

I/O 设备 (Input/Output Devices)

  • 用于计算机与外部世界进行交互。通过 I/O 接口与主机相连。

3. 计算机性能指标

衡量一台计算机性能的好坏，通常有以下几个关键指标：

  • 吞吐量 (Throughput): 表示在单位时间内系统处理的请求数量。它衡量的是系统的处理能力，对于服务器等批处理应用尤为重要。

  • 响应时间 (Response Time): 指从用户发送一个请求到系统给出响应所花费的时间。它衡量的是系统的即时响应能力，对于交互式应用（如网站、桌面应用）至关重要。

  • 时钟周期 (Clock Cycle): 是计算机中最基本的时间单位，由 CPU 的主频决定。主频越高，时钟周期越短。

  • CPI (Cycles Per Instruction): 表示执行一条指令所需的平均时钟周期数。CPI 越小，表示执行指令所需的时间越短。

  • IPS (Instructions Per Second): 表示每秒钟 CPU 能执行的指令条数。IPS = 主频 / CPI。

  • MIPS (Million Instructions Per Second): 每秒执行百万条指令。这是一个更常用的衡量 CPU 速度的单位。

  • FLOPS (Floating-point Operations Per Second): 每秒执行的浮点运算次数。这是衡量科学计算和图形处理能力的重要指标。

CPU 执行时间公式: CPU时间 = 指令数 × CPI × 时钟周期

这个公式是计算机性能分析的核心。要提高 CPU 性能，可以从三个方面入手：

  1. 减少指令数: 优化算法和编译器。
  2. 降低 CPI: 改进指令集和 CPU 的微架构（如使用流水线技术）。
  3. 缩短时钟周期: 提高 CPU 主频（受限于物理工艺和功耗）。
