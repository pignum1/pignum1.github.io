计算机网络 - 网络协议分层模型

为了设计和实现复杂的计算机网络，人们提出了分层（Layering）的思想。分层将一个复杂的问题分解为多个更小、更易于管理的部分。每一层都建立在它的下层之上，并为它的上层提供特定的服务，同时隐藏下层实现的细节。网络协议分层模型主要有两种：OSI 七层模型和 TCP/IP 四层/五层模型。

1. OSI 七层参考模型 (Open Systems Interconnection)

OSI 模型是一个理论上的、非常完善的模型，由国际标准化组织（ISO）提出。它旨在为所有计算机网络提供一个标准的参考框架。

层次 名称 功能描述 常见协议/设备
第7层 应用层 (Application) 为应用程序提供网络服务，是用户直接接触的一层。 HTTP, FTP, SMTP, DNS
第6层 表示层 (Presentation) 负责数据的格式化、加密/解密、压缩/解压缩。 JPEG, ASCII, TLS/SSL
第5层 会话层 (Session) 建立、管理和终止会话（连接）。 NetBIOS, RPC
第4层 传输层 (Transport) 提供端到端（End-to-End）的可靠或不可靠的数据传输服务。 TCP, UDP
第3层 网络层 (Network) 负责逻辑地址（IP地址）寻址和路由选择，将数据包从源头送到目的地。 IP, ICMP, OSPF
第2层 数据链路层 (Data Link) 负责物理地址（MAC地址）寻址，在相邻节点间传输数据帧，并进行差错校验。 Ethernet, PPP
第1层 物理层 (Physical) 负责在物理媒介（如网线、光纤）上传输原始的比特流。 RJ45, 光纤

2. TCP/IP 模型

TCP/IP 模型是事实上的工业标准，是当前互联网所使用的模型。它比 OSI 模型更简洁、更实用。

TCP/IP 四层模型

这是 TCP/IP 协议的经典划分。

层次 名称 对应 OSI 模型 功能描述
第4层 应用层 (Application) 应用层, 表示层, 会话层 处理特定应用程序的协议，如 HTTP, DNS 等。
第3层 传输层 (Transport) 传输层 提供端到端的通信，主要是 TCP 和 UDP。
第2层 网络层 (Internet) 网络层 负责数据包的路由和转发，核心是 IP 协议。
第1层 网络接口层 (Network Interface) 数据链路层, 物理层 负责与物理网络（如以太网）的接口。

TCP/IP 五层模型

为了更好地教学和理解，人们通常将 TCP/IP 的网络接口层再细分为数据链路层和物理层，从而形成一个五层模型。这个模型在功能划分上与 OSI 模型更为接近，也更容易理解。

3. 数据封装与解封装过程

当应用程序发送数据时，数据会自顶向下穿过协议栈的每一层，每一层都会在前一层数据的基础上添加自己的头部信息（Header），这个过程称为封装 (Encapsulation)。

  1. 应用层: 原始数据（如 HTTP 请求）
  2. 传输层: 添加 TCP/UDP 头部（包含端口号等），形成 TCP/UDP 报文段 (Segment)。
  3. 网络层: 添加 IP 头部（包含源/目的 IP 地址等），形成 IP 数据包 (Packet)。
  4. 数据链路层: 添加 MAC 头部（包含源/目的 MAC 地址等），形成 帧 (Frame)。
  5. 物理层: 将帧转换为二进制比特流，通过物理介质传输。

当数据到达接收方时，会发生一个相反的过程——解封装 (De-encapsulation)。数据自底向上穿过协议栈，每一层都会解析并移除相应协议的头部信息，最终将原始数据还原给应用程序。

总结

  • OSI 模型是一个理论完备的参考模型，用于指导和标准化协议的设计。
  • TCP/IP 模型是互联网的事实标准，它更简洁、更高效。
  • 分层是网络设计的核心思想，它使得网络的不同部分可以独立开发和演进，同时保证了互操作性。
