并发编程 - java.util.concurrent (JUC) 核心工具

java.util.concurrent 包（简称 JUC）是 Java 并发编程的核心，由并发大师 Doug Lea 设计，提供了比 synchronized 和 volatile 更高级、更丰富的并发工具。掌握 JUC 是成为 Java 高级工程师的必备技能。

1. 线程池 (ExecutorService)

线程池用于统一管理和复用线程，避免因频繁创建和销毁线程而带来的性能开销。

  • 核心实现: ThreadPoolExecutor。
  • 7大核心参数: corePoolSize, maximumPoolSize, keepAliveTime, unit, workQueue, threadFactory, handler。
  • 工作原理: 核心线程 -> 任务队列 -> 最大线程 -> 拒绝策略。
  • Executors 风险: newFixedThreadPool 和 newCachedThreadPool 等预设线程池可能因使用无界队列或无界最大线程数而导致 OOM，生产环境建议手动创建 ThreadPoolExecutor。
  • 配置原则: CPU 密集型任务配置 CPU核心数+1 个线程；I/O 密集型任务配置更多线程。

2. 并发容器 (Concurrent Containers)

JUC 提供了一系列线程安全的容器类，性能远超使用 Collections.synchronized* 包装的传统容器。

2.1 ConcurrentHashMap 深度解析

  • 核心优势: 保证线程安全的同时，提供极高的并发性能。
  • JDK 1.8 实现:
    • 数据结构: Node[] + 链表/红黑树。
    • 锁机制: CAS + synchronized 锁节点。put 在哈希冲突时才加锁，get 全程不加锁。
    • size() 计数: baseCount + CounterCell 数组，无锁高效计数。

2.2 CopyOnWriteArrayList (写时复制列表)

  • 核心思想: 写时复制 (Copy-On-Write)。读写分离。
  • 读操作: 不加锁，性能极高。
  • 写操作: 加锁并复制整个数组，在副本上修改，然后替换引用。
  • 适用场景: 读多写少的并发场景（如系统配置、监听器列表）。

2.3 BlockingQueue (阻塞队列)

  • 核心特性: 队列为空时 take() 阻塞；队列满时 put() 阻塞。
  • 常用实现:
    • ArrayBlockingQueue: 基于数组的有界队列。
    • LinkedBlockingQueue: 基于链表的无界（默认）队列。
    • SynchronousQueue: 不存储元素的“手递手”队列。
    • PriorityBlockingQueue: 支持优先级的无界队列。

3. 并发同步工具 (Synchronizers)

JUC 提供了一系列用于线程协同工作的同步工具。

3.1 CountDownLatch (倒计时门闩)

CountDownLatch 允许一个或多个线程等待其他一组线程完成操作。

  • 核心概念:
    • 计数器 (count): 创建时指定一个初始计数值。
    • await(): 调用此方法的线程会被阻塞，直到计数器值为 0。
    • countDown(): 使计数器的值减 1，不阻塞。
  • 工作原理: 基于 AQS 的共享模式实现。
  • 应用场景: 主线程等待所有子任务完成；实现最大并发性测试。
  • 注意: CountDownLatch 是一次性的，计数器值无法重置。

3.2 Semaphore (信号量)

Semaphore 用于控制同时访问特定资源的线程数量，常用于实现流量控制、连接池等场景。

  • 核心概念:
    • 许可证 (permits): 创建时指定一个初始的“许可证”数量。
    • acquire(): 获取一个许可证。如果无可用许可证，线程将阻塞。
    • release(): 释放一个许可证，将其归还给 Semaphore。
  • 工作原理: 基于 AQS 实现，state 的值代表当前可用的许可证数量。
  • 公平性: 支持公平模式（等待最久的线程优先）和非公平模式（允许插队，默认）。

3.3 CyclicBarrier (循环栅栏)

CyclicBarrier 的作用是让一组线程相互等待，直到所有线程都到达一个公共的屏障点 (Barrier Point)，然后再一起继续执行。

  • 核心概念:
    • 参与方数量 (parties): 创建时指定等待的线程数量。
    • await(): 线程调用此方法来通知栅栏“我到了”，然后被阻塞。
    • 栅栏动作 (barrierAction): (可选) 当所有线程都到达栅栏后，在释放它们之前，会由其中一个线程先执行这个 Runnable 任务。
  • CyclicBarrier vs CountDownLatch:
    特性 CyclicBarrier (循环栅栏) CountDownLatch (倒计时门闩)
    作用对象 线程。让一组线程相互等待。 事件。让一个或多个线程等待其他线程完成事件。
    核心方法 await() countDown() 和 await()
    可重用性 可重用。栅栏被打破后，可以自动或手动重置。 一次性。计数器减到 0 后无法重置。
  • 简单区分: CountDownLatch 是“减法”，一个线程等多个；CyclicBarrier 是“加法”，多个线程等彼此。
