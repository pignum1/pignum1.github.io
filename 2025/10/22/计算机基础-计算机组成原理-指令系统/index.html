指令系统 (Instruction Set Architecture - ISA)

指令系统（ISA）是计算机体系结构中与程序设计有关的部分，它是软件和硬件之间的关键接口。它定义了 CPU 能识别和执行的指令集合，规定了指令的格式、寻址方式以及指令的功能。

1. 指令格式

一条指令就是机器语言的一个语句，它规定了计算机要执行的某种操作。一条指令通常由两部分组成：

  • 操作码 (Opcode): 指明该指令要执行的操作，如加、减、取数、存数等。操作码的位数决定了计算机的指令集大小（例如，n 位操作码最多可以表示 2^n 条指令）。
  • 地址码 (Address Code): 指明操作的对象（操作数）的地址。根据地址码的数量，指令可以分为零地址、一地址、二地址、三地址指令。
操作码 (Opcode) 地址码 1 (A1) 地址码 2 (A2) …
  • 三地址指令: OP A1, A2, A3 -> (A1) OP (A2) -> A3 (例如，ADD R1, R2, R3)
  • 二地址指令: OP A1, A2 -> (A1) OP (A2) -> A1 (例如，ADD R1, R2)
  • 一地址指令: OP A1 -> (ACC) OP (A1) -> ACC (需要一个隐含的累加器 ACC)
  • 零地址指令: 无地址码，操作数隐含在栈顶。常用于堆栈式计算机。

2. 指令类型

指令按功能可以大致分为以下几类：

  • 数据传送指令: 在寄存器之间、寄存器与内存之间、内存与内存之间传送数据。例如 LOAD (取数), STORE (存数), MOVE (移动)。
  • 算术逻辑指令: 执行算术运算（加、减、乘、除）和逻辑运算（与、或、非、异或）。
  • 移位指令: 对操作数进行算术移位、逻辑移位或循环移位。
  • 控制转移指令: 控制程序的执行流程。
    • 无条件转移: JMP (跳转)
    • 条件转移: JZ (结果为零则跳转), JNZ (结果非零则跳转)
    • 过程调用与返回: CALL, RET

3. 寻址方式 (Addressing Modes)

寻址方式是指指令中如何指定操作数的位置。CPU 根据指令中的寻址方式来找到操作数的实际地址（有效地址）。

  • 立即寻址 (Immediate Addressing)

    • 定义: 地址码字段直接给出了操作数本身，而不是操作数的地址。
    • 优点: 获取操作数快，无需访存。
    • 缺点: 操作数大小受地址码字段长度限制。
    • 示例: ADD R1, #5 (将寄存器 R1 的内容与立即数 5 相加)

  • 直接寻址 (Direct Addressing)

    • 定义: 地址码字段给出的是操作数在内存中的有效地址。
    • 优点: 简单直观。
    • 缺点: 寻址范围受地址码字段长度限制。
    • 示例: LOAD R1, 1000H (将内存地址 1000H 处的数据加载到 R1)

  • 间接寻址 (Indirect Addressing)

    • 定义: 地址码字段给出的是一个内存地址，该地址中存放的是操作数的有效地址。
    • 优点: 扩大了寻址范围。
    • 缺点: 需要多次访存，速度较慢。
    • 示例: LOAD R1, @1000H (先读取内存 1000H 的内容，得到有效地址 EA，再从 EA 读取数据到 R1)

  • 寄存器寻址 (Register Addressing)

    • 定义: 地址码字段指定的是一个寄存器编号，操作数就在该寄存器中。
    • 优点: 速度极快，因为无需访存。
    • 缺点: 寄存器数量有限。
    • 示例: ADD R1, R2 (将 R1 和 R2 的内容相加)

  • 寄存器间接寻址 (Register Indirect Addressing)

    • 定义: 地址码字段指定一个寄存器，该寄存器中存放的是操作数的有效地址。
    • 优点: 比内存间接寻址快，因为只需访存一次。
    • 示例: LOAD R1, (R2) (将 R2 寄存器内容作为地址，读取内存数据到 R1)

  • 变址寻址 (Indexed Addressing)

    • 定义: 有效地址等于地址码字段给出的基地址与一个变址寄存器内容的和。
    • 优点: 非常适合处理数组和循环。只需改变变址寄存器的值，就可以方便地访问数组中的每个元素。
    • 示例: LOAD R1, 100(R2) (有效地址 = 100 + (R2))

4. RISC vs. CISC

这是两种主流的指令集设计哲学。

CISC (Complex Instruction Set Computer - 复杂指令集计算机)

  • 思想: 试图用一条指令完成一个复杂的功能。
  • 特点:
    • 指令数量多，功能复杂。
    • 指令长度不固定，寻址方式多样。
    • 通常采用微程序控制器。
  • 代表: Intel x86 架构（我们日常使用的 PC 的 CPU）。

RISC (Reduced Instruction Set Computer - 精简指令集计算机)

  • 思想: 简化指令集，降低硬件设计复杂度，通过优化编译器来完成复杂功能。
  • 特点:
    • 指令数量少，格式统一，长度固定。
    • 只提供最常用的指令，复杂操作由简单指令组合而成。
    • 寻址方式简单，大部分操作在寄存器之间进行。
    • 通常采用硬布线控制器和流水线技术。
  • 代表: ARM 架构（绝大多数手机和平板的 CPU）、MIPS、RISC-V。

指令长度 可变 固定

寄存器 少 多
