JavaSE - Java 8+ 新特性

Java 8 是自 Java 5 以来最重要的一次版本更新，它引入了函数式编程的思想，极大地改变了 Java 的编程风格，使代码更简洁、更具表达力。本笔记将重点介绍其最核心的几个新特性。

1. Lambda 表达式

Lambda 表达式是 Java 8 的核心特性之一，它允许我们将函数作为方法的参数，或者说，将代码块视为数据。其本质是一个匿名函数。

  • 作用: 极大地简化了匿名内部类的写法，尤其是在使用函数式接口时。
  • 语法: (parameters) -> expression 或 (parameters) -> { statements; }

函数式接口 (Functional Interface)

Lambda 表达式能够被赋值给一个变量，这个变量的类型必须是函数式接口。

  • 定义: 只包含一个抽象方法的接口。
  • 注解: @FunctionalInterface 注解用于在编译时检查一个接口是否满足函数式接口的定义。
  • Java 内置函数式接口: java.util.function 包提供了大量常用的函数式接口：
    • Predicate<T>: 接收一个参数，返回 boolean 值 (e.g., t -> t > 0)。
    • Consumer<T>: 接收一个参数，无返回值 (e.g., t -> System.out.println(t))。
    • Function<T, R>: 接收一个参数，返回一个结果 (e.g., t -> t.length())。
    • Supplier<T>: 不接收参数，返回一个结果 (e.g., () -> new String("hello"))。

2. Stream API：像工厂流水线一样处理数据

Stream API 是对集合（Collection）功能的巨大增强，它专注于对集合对象进行各种高效的聚合操作或批量操作。为了彻底理解它，我们可以把它想象成一条工厂的流水线。

Stream 的直观理解：一条工厂流水线

你的集合（比如一个 List）就是装满了原材料的箱子。你的目标是对这箱原材料进行一系列加工，最终得到成品。

这个过程分为三个步骤：

  1. 把原材料放到流水线上 (创建 Stream)

    • 你不能直接在箱子里加工原材料，必须先把它们一个个放到流水线上。
    • 这个动作就是 list.stream()。它创建了一个流，但此时不会进行任何加工。

  2. 流水线上的加工站 (中间操作 - Intermediate Operations)

    • 流水线上有很多个加工站，每个站只负责一个特定的任务。
    • filter() (筛选站): 就像一个筛子，不符合条件的原材料会从流水线上掉下去。
    • map() (变形站): 对通过的原材料进行变形（例如，把字符串变成它的长度，或转为大写）。
    • sorted() (排序站): 把流水线上的原材料按照顺序排好。
    • distinct() (去重站): 如果流水线上有重复的原材料，这个站会把多余的扔掉，只保留一个。

    关键特性：惰性求值 (Laziness)

    • 这些加工站非常“懒”。当你把它们设置好后（即写好 filter, map 等代码），它们并不会立即开始工作。
    • 它们只是知道了自己的任务是什么，然后就等着“开工”的信号。这就像你已经设计好了流水线的所有工序，但还没有按下启动按钮。

  3. 打包成品 (终端操作 - Terminal Operations)

    • 这是流水线的最后一个环节，也是触发整个流水线启动的信号。
    • collect(Collectors.toList()) (装箱站): 把流水线上所有加工好的成品，收集起来，装到一个新的箱子（List）里。
    • forEach() (检查站): 不装箱，而是把每个成品拿出来看一眼（比如打印出来）。
    • count() (计数站): 不关心成品长什么样，只数一数最终有多少个成品。

    关键特性：触发执行 & 一次性

    • 一旦你设置了终端操作（比如 collect），就相当于按下了流水线的启动按钮。此时，原材料才会真正地从箱子里出来，一个接一个地流过所有的中间加工站，最后在终端站完成打包。
    • 一条流水线（一个 Stream）在完成一次打包后，就不能再用了。如果你想再加工一次，必须从箱子里重新取原材料，再开一条新的流水线（list.stream()）。

Stream 的核心思想总结

  • 声明式编程: 你只关心**“做什么”（筛选、变形、排序），而不关心“怎么做”**（不需要自己写 for 循环、if 判断）。
  • 管道化与惰性求值: 中间操作会连接起来形成一条管道，只有在终端操作被调用时，数据才会在管道中流动。这种机制使得 Stream API 可以在内部进行很多优化。
  • 无副作用: Stream 操作不修改源数据，总是返回一个新的 Stream 或结果，这使得代码更安全、更易于并行处理。

并行流 (Parallel Stream)

通过 .parallelStream() 或 .stream().parallel() 可以轻松地创建并行流，它会自动利用多核 CPU 并行处理数据，在处理大量数据时能显著提升性能。

注意: 使用并行流时，要确保操作是线程安全的。

3. Optional 类

Optional 是一个可以为 null 的容器对象，旨在优雅地处理 null 值，避免 NullPointerException (NPE)。

  • 核心思想: 显式地提醒调用者“这里可能没有值”，强制开发者处理 null 的情况。

创建 Optional 对象

  • Optional.of(value): 创建一个包含非 null 值的 Optional。如果 value 为 null，会立即抛出 NullPointerException。
  • Optional.ofNullable(value): 创建一个 Optional，允许 value 为 null。
  • Optional.empty(): 创建一个空的 Optional。

5. 其他重要新特性

除了以上四大核心特性，Java 8 还引入了许多其他同样重要的新功能。

5.1 接口的默认方法和静态方法

  • 为什么需要: 在 Java 8 之前，如果想给一个接口增加新方法，所有实现了该接口的类都必须重写这个新方法，这对于已有的庞大类库（如 java.util.List）来说是灾难性的。
  • 默认方法 (default):
    • 允许在接口中提供一个方法的默认实现。
    • 实现该接口的类可以不重写这个默认方法，直接使用它。
    • 作用: 解决了接口的向后兼容性问题。List 接口的 sort() 方法、forEach() 方法就是默认方法。
  • 静态方法 (static):
    • 允许在接口中定义静态方法。
    • 这些方法只能通过接口名直接调用，不能被实现类继承或重写。
    • 作用: 通常用于定义与接口相关的工具方法，避免创建多余的工具类（如 Collections 类）。

5.2 方法引用 (Method References)

方法引用是 Lambda 表达式的一种语法糖，它可以让代码看起来更简洁、更易读。当你想要执行的 Lambda 操作只是调用一个已存在的方法时，就可以使用方法引用。

类型 示例 Lambda 表达式 方法引用
静态方法引用 Integer.parseInt(s) s -> Integer.parseInt(s) Integer::parseInt
实例方法引用 (特定对象) System.out.println(s) s -> System.out.println(s) System.out::println
实例方法引用 (任意对象) s.toUpperCase() s -> s.toUpperCase() String::toUpperCase
构造方法引用 new ArrayList<>() () -> new ArrayList<>() ArrayList::new

5.3 CompletableFuture (异步编程)

CompletableFuture 是对 Future 的巨大增强，它为异步编程提供了一套强大的、函数式的 API，可以轻松地组合、链接和处理多个异步任务的结果，而无需陷入“回调地狱”。

5.4 Base64 编码支持

Java 8 在 java.util 包中增加了 Base64 类，提供了官方的、高性能的 Base64 编码和解码支持，无需再依赖第三方库。
