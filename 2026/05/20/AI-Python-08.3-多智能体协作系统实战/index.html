AI-Python-08.3: 多智能体协作系统实战

前面我们学会了构建单个 Agent（08.2），但现实中的复杂任务往往需要多个 Agent 协作。本文从三种最实用的协作模式入手，用 LangGraph StateGraph 构建完整的多 Agent 系统。

前置知识：

  • 06.6 LangGraph 状态机编排：StateGraph 基础 + Supervisor 概念
  • 08.1 Agent 设计模式全景：15 种模式理论（含 6 种协作模式）
  • 08.2 Agent 实战：单 Agent 的完整构建过程

1. 从单 Agent 到多 Agent

1.1 单 Agent 的瓶颈

当任务变复杂，单 Agent 会遇到三个问题：

问题 原因 表现
角色混乱 一个 Agent 既要搜索又要分析又要写作 Prompt 太长，LLM 不知道优先做什么
工具爆炸 搜索 + 阅读 + 计算 + 翻译全塞一起 LLM 选错工具、参数幻觉
无法并行 一个 Agent 只能串行处理 3 个独立任务也要排队

1.2 多 Agent 的核心：分工 + 协调

多 Agent 的设计只有两个关键问题：

  1. 分工：每个 Agent 专注一件事（单一职责）
  2. 协调：谁来分配任务、怎么传递信息

1.3 三种实战模式

本文重点实现三种最实用的协作模式：

模式 核心思想 适用场景
Supervisor 主管分配，专家执行 角色明确的协作
Pipeline 固定流水线，依次加工 流程固定的任务链
Map-Reduce 并行调研，汇总对比 独立子任务

2. 实战 1：Supervisor 模式 — 知识库研究团队

2.1 模式说明

一个”主管 Agent”分析用户意图，把任务分发给最合适的”专家 Agent”。每个专家只做自己擅长的事，做完后回到主管，由主管决定下一步。

2.2 关键设计

三个设计要点：

  1. Supervisor 做”选择题” — 不是自由文本，而是让 LLM 从 researcher / analyst / writer / FINISH 中选一个
  2. called_agents 防重复 — 记录已调过的专家，不重复调用
  3. round_count 防死循环 — 超过最大轮数强制交给 Writer 收尾

4. 实战 3：并行 Map-Reduce — 竞品分析

4.1 模式说明

分而治之：拆分任务 → 并行执行 → 汇总结果。

LangGraph 并行的关键：START → 多个节点 = 自动并行，汇总节点自动等待所有 Worker 完成。

4.5 并行的原理

LangGraph 的并行是节点级别的：当一个节点有多个后继，且这些后继不依赖彼此，就会并行执行。汇总节点（fan-in）会自动等待所有上游节点完成。

完整可运行代码：exercise_07_mapreduce.py

5. 三种模式对比与选型

5.1 全面对比

维度 Supervisor Pipeline Map-Reduce
流程 动态（LLM 路由） 固定（顺序） 并行 → 汇总
LLM 调用 每次 Supervisor 决策都调 只在节点内调 每个 Worker 独立调
并行 不支持 不支持 原生支持
回退/循环 支持（回到 Supervisor） 支持（条件边回退） 不支持
调试难度 中（需看路由） 低（流程确定） 低（互相独立）
Token 消耗 较高（路由开销） 较低 取决于 Worker 数

5.2 选型决策

  • 角色不固定、需要灵活调度 → Supervisor
  • 流程固定、有质量检查 → Pipeline
  • 子任务独立、需要并行 → Map-Reduce
  • 复杂项目 → 组合使用（Supervisor 管理多个 Pipeline）

5.3 生产经验

从 Supervisor 开始。它是最通用的模式，能覆盖 80% 的场景。复杂了再拆成 Pipeline 或 Map-Reduce。不要一上来就搞 Hierarchical（层级模式），调试成本太高。

6. 实战练习

练习 1（入门）：改造 Supervisor

在现有的 Supervisor 团队中增加一个”翻译专家”Agent：

  • Translator：将中文回答翻译成英文
  • 修改 Supervisor 的路由逻辑，在 Writer 之后可选调用 Translator

练习 2（进阶）：Hierarchical 模式

实现层级模式（CEO → CTO/PM → Workers）：

  • CEO 分配任务给 CTO 或 PM
  • CTO 管理后端/前端 Agent
  • PM 管理设计/测试 Agent
  • 提示：用嵌套的 StateGraph 实现

练习 3（挑战）：Supervisor + Map-Reduce 组合

实现一个混合模式：

  • Supervisor 分析任务，决定是否需要并行
  • 需要并行时启动 Map-Reduce 子图
  • 不需要时走普通 Pipeline

7. 知识图谱

学习路径：单 Agent → 多 Agent 协作 → Supervisor/Pipeline/Map-Reduce → 组合模式

模式层级：Pipeline（固定流程）< Supervisor（动态路由）< Hierarchical（多层嵌套）
