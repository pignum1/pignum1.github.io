AI-Python-07.2: 文档智能解析

RAG 的第一步是把各种格式的文档变成”可检索的文本”。本章深入 PDF 解析、OCR、表格提取、布局分析，以及 LangChain 文档加载器的实战用法。

1. 为什么文档解析是 RAG 的瓶颈？

1.1 真实文档的复杂性

RAG 系统面对的不是干净的文本文件，而是：

文档类型 解析难点
PDF 混合排版、多栏布局、嵌入图片、扫描件
Word/Excel 表格结构、样式信息、公式
图片/PPT 纯视觉内容，需要 OCR
网页 HTML 标签、广告、导航栏干扰 
    flowchart LR
    A[原始文档] --> B{文档类型?}
    B -->|纯文本| C[直接读取]
    B -->|PDF 数字版| D[文本提取]
    B -->|PDF 扫描件| E[OCR 识别]
    B -->|图片| E
    B -->|网页| F[HTML 清洗]

    C --> G[统一 Document 对象]
    D --> G
    E --> G
    F --> G

    style G fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333

1.2 解析质量直接决定 RAG 效果

垃圾进，垃圾出（Garbage In, Garbage Out）——文档解析是 RAG 质量的地基。

2.1 常用 Loader 一览

Loader 数据源 输出

PyPDFLoader PDF 文件 每页一个 Document
Docx2txtLoader Word 文档 Document
CSVLoader CSV 表格 每行一个 Document
WebBaseLoader 网页 Document
DirectoryLoader 整个文件夹 多个 Document
UnstructuredLoader 任意格式（万能） Document

3. PDF 解析三大方案对比

3.1 方案对比

方案 原理 优点 缺点
PyPDF2 / pypdf 提取 PDF 内嵌文本 快、轻量 扫描件不行、表格乱码
Marker 模型驱动的布局分析 高质量 Markdown 输出 较慢、需 GPU
Docling IBM 开源文档处理 表格/公式支持好 依赖较多

3.2 PyPDFLoader（基础方案）

适合数字 PDF（文字可选中复制的）：

局限： 遇到扫描件（图片型 PDF）提取不到文字，遇到多栏排版会把两栏文字交叉混在一起。

3.3 Marker（高质量方案）

Marker = 深度学习模型驱动的 PDF → Markdown 转换工具

核心能力：

  • 识别文档布局（标题、段落、列表、代码块）
  • 保留表格结构（输出 Markdown 表格）
  • 处理多栏排版
  • 提取图片中的文字

Marker 输出示例：

3.4 Docling（IBM 开源方案）

Docling 的优势：

  • 学术论文支持好（公式、引用、脚注）
  • 表格结构提取准确
  • 支持 PDF、Word、PPT、HTML、图片等多种格式
