AI-Python-07.4: GraphRAG 知识图谱检索

传统 RAG 只能检索”文本片段”，无法理解实体间的关系。GraphRAG 将知识图谱与向量检索结合，能回答”谁和谁有什么关系”这类需要跨文档推理的问题。本章讲解知识图谱构建、图谱检索与向量检索的融合。

1. 传统 RAG 的局限

1.1 什么问题传统 RAG 回答不了？

场景： 知识库里有 10 篇关于公司员工的文档。

根因： 传统 RAG 只能找”写出来的文本”，无法推理”没写出来但可以从关系推导的信息”。

1.2 GraphRAG 如何解决

GraphRAG 在向量检索之外，额外构建一个知识图谱，显式存储实体和关系：

    flowchart LR
    A[张三] -->|汇报给| B[王五]
    C[李四] -->|汇报给| B
    A -->|所属| D[技术部]
    C -->|所属| E[市场部]
    B -->|职位| F[CTO]
    A -->|职位| G[经理]
    C -->|职位| H[总监]

    style C fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333

用户问”张三和李四的关系”时，图谱可以沿着边遍历，推理出两人都汇报给王五。

2. 知识图谱基础

2.1 什么是知识图谱？

知识图谱 = 实体（节点）+ 关系（边）+ 属性

组成 例子 说明
实体（Node） 张三、技术部、Python 图中的”点”
关系（Edge） 汇报给、所属、擅长 连接两个实体的”线”
属性（Property） 年龄=30、级别=P7 实体上的附加信息

2.3 知识图谱 vs 向量检索

维度 向量检索 知识图谱
存储 文本片段 + 向量 实体 + 关系
擅长 “X是什么？” “X和Y有什么关系？”
推理 无（只匹配相似文本） 有（沿关系遍历）
精确度 模糊匹配 精确结构化
构建成本 低（自动 Embedding） 高（需要抽取实体和关系）

4. 图数据库：Neo4j

4.1 为什么用 Neo4j？

Neo4j 是最流行的图数据库，天然支持实体和关系的存储与查询。

操作 Cypher 语句
创建节点 CREATE (n:Person {name: "张三"})
创建关系 MATCH (a),(b) CREATE (a)-[:REPORTS_TO]->(b)
查询关系 MATCH (a)-[r]->(b) WHERE a.name="张三" RETURN r,b

5. GraphRAG：图谱 + 向量融合

5.1 为什么要融合？

单独用 能回答 不能回答
向量检索 “Python装饰器怎么用？” “张三和李四是什么关系？”
知识图谱 “张三和李四是什么关系？” “请详细解释FlashAttention的原理”

融合后： 两种问题都能回答。

5.2 融合架构

    flowchart TB
    Q[用户问题] --> C{问题类型?}

    C -->|事实查询\nX是什么| V[向量检索]
    C -->|关系查询\nX和Y的关系| G[图谱查询]
    C -->|复杂推理| H[混合检索\n向量+图谱]

    V --> L[LLM 生成回答]
    G --> L
    H --> L

6. GraphRAG 的适用场景

6.1 什么时候用 GraphRAG？

场景 是否需要 GraphRAG 原因
企业组织关系 是 需要推理汇报链、协作关系
人物/事件关联分析 是 需要跨文档连接信息
知识推理 是 需要多跳推理（A→B→C）
简单 FAQ 否 向量检索足够
文档问答 否 不涉及关系推理
代码/技术文档 否 查具体概念即可

6.2 GraphRAG 的成本

维度 成本
构建 高（需要 LLM 抽取实体/关系）
存储 中（图数据库 + 向量数据库）
查询 中（需要判断路由 + 两路查询）
维护 高（实体/关系需要更新）

建议： 只有确实需要”关系推理”的场景才上 GraphRAG，普通 RAG 场景用 07.1-07.3 的方案就够了。

7. 速查表

知识图谱速查

操作 Cypher 语句
创建节点 MERGE (n:Label {name: "xxx"})
创建关系 MATCH (a),(b) MERGE (a)-[:REL]->(b)
查 1 跳 MATCH (a)-[r]->(b) WHERE a.name="X" RETURN b
查 2 跳 MATCH (a)-[r*1..2]-(b) WHERE a.name="X" RETURN b

RAG 方案选择

需求 方案
简单文档问答 向量检索 RAG（07.1）
精确 + 语义 混合检索 + Rerank（07.3）
关系推理 GraphRAG（本章）
全都要 混合检索 + GraphRAG
