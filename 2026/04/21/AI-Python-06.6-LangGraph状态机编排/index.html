AI-Python-06.6: LangGraph 状态机编排

LangChain Agent 是”黑盒”，LLM 自己决定一切。LangGraph 把 Agent 拆成”白盒”——每个步骤都是显式的节点，节点之间用明确的规则连接。本章深入理解 State、Node、Edge 和多 Agent 协作模式。

1.2 LangGraph 的解决思路

把 Agent 拆成状态机——每个步骤是节点，节点之间是明确的规则：

    flowchart LR
    subgraph LangChain Agent 黑盒

4. 三种 Agent 决策模式

ReAct 是最经典的模式，但 Agent 还有其他决策方式，每种适合不同的场景。

4.1 ReAct（边想边做）

每一步临时决定下一步做什么，根据上一步的结果灵活调整。

    flowchart TB
    S[START] --> A[agent\nThought 思考]
    A --> C{有 tool_calls?}
    C -->|是| T[tools\nAction + Observation]
    T --> A
    C -->|否| E[END\nFinal Answer]

    style A fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333
    style T fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333

适用场景： 不确定需要几步的任务（通用场景，90% 情况够用）。

4.2 Plan-and-Execute（先规划再执行）

先把所有步骤想好，再逐步按计划执行。

    flowchart TB
    S[START] --> P[Planner\n规划全部步骤]
    P --> E1[Executor\n执行步骤1]
    E1 --> C1{还有步骤?}
    C1 -->|是| E2[Executor\n执行步骤2]
    E2 --> C2{还有步骤?}
    C2 -->|否| SUM[Summarizer\n汇总报告]
    C1 -->|否| SUM
    SUM --> E[END]

    style P fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333
    style E1 fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333
    style E2 fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333
    style SUM fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333

适用场景： 步骤明确的大型任务（如”做一份技术调研报告”）。

4.3 Reflexion（自我反思）

做完之后自我检查，发现问题就修复，直到通过。

    flowchart TB
    S[START] --> G[Generator\n生成代码]
    G --> R[Reviewer\n自我审查]
    R --> C{审查通过?}
    C -->|FAIL| G
    C -->|PASS| F[Final\n输出结果]
    F --> E[END]

    style G fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333
    style R fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333

适用场景： 容易出错、需要验证的任务（如代码生成、数学推理）。

4.4 三种模式对比

维度 ReAct Plan-and-Execute Reflexion
策略 边想边做 先规划再执行 做完再检查
图结构 agent → tools 循环 planner → executor 循环 generator → reviewer 循环
灵活性 最高 最低 中等
Token 消耗 中等 较少 最多（多轮审查）
准确率 中等 看规划质量 最高（有自检）
适用场景 通用、不确定的任务 步骤明确的任务 容易出错的任务

9. LangGraph vs LangChain Agent

维度 LangChain Agent LangGraph
流程控制 黑盒，LLM 自己决定 白盒，你定义每一步
循环 隐式（ReAct 循环） 显式（add_edge 回到之前的节点）
人工审批 不支持 原生支持 interrupt_before
状态管理 无 State + Checkpointer
调试 困难 每个节点可单独调试
多 Agent 不支持 Supervisor / Swarm / 并行
适用场景 简单问答 复杂工作流、生产环境

LCEL 在 LangGraph 中的角色： LangGraph 管节点之间的流转（宏观），LCEL 管节点内部的数据处理（微观）。
