AI-Python-06.3: LCEL 链式编排

LCEL（LangChain Expression Language）是 LangChain 的核心语法，使用 | 管道操作符串联组件。本章深入理解 Runnable 接口、四种调用方式、三个工具类和高级编排模式。

1. 为什么需要 LCEL？

1.1 没有 LCEL 的写法

1.2 有了 LCEL

LCEL 解决的核心问题： 每个组件的输入输出格式不同，手动传递容易出错。

    flowchart LR
    A[PromptTemplate\n输出: list-BaseMessage] --> B[ChatOpenAI\n输出: AIMessage]
    B --> C[OutputParser\n输出: str/dict/object]

    style A fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333
    style B fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333
    style C fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333

LCEL 自动处理类型转换，开发者只需关注业务逻辑。

3. Runnable 接口：一切皆可管道

LangChain 中几乎所有组件都实现了 Runnable 接口：

    R --> D[RunnablePassthrough\n原样传递]
    R --> E[RunnableLambda\n包装函数]
    R --> F[RunnableParallel\n并行执行]

任何 Runnable 都有相同的 4 种调用方式，这就是 LCEL 的统一性。

4. 四种调用方式

4.1 invoke — 单次调用

最基础的调用方式，返回完整结果。

4.2 batch — 批量调用

一次发送多个请求，返回结果列表。

参数 说明
inputs 输入列表
config.max_concurrency 最大并发数

4.3 stream — 流式输出

逐 token 输出，像 ChatGPT 那样一个字一个字出现。

流式输出的数据流：

    sequenceDiagram
    participant U as 用户
    participant P as Prompt
    participant L as LLM
    participant S as Stream

    U->>P: invoke 一次性生成
    P->>L: 消息列表
    loop 逐 token
        L->>S: AIMessageChunk
        S->>U: 打印 chunk
    end

注意： Prompt 不是流式的（一次性生成消息列表），只有 LLM 是流式输出。

4.4 异步方法

异步在并发请求和 Web 服务（如 FastAPI 集成）中非常重要。

4.5 四种方式对比

方法 同步 异步 返回类型 适用场景
invoke .invoke() .ainvoke() 完整结果 单次请求
batch .batch() .abatch() 结果列表 批量处理
stream .stream() .astream() 迭代器 实时输出、聊天界面
事件流 - .astream_events() 事件流 精细控制、调试

5. 三个核心工具类

5.1 RunnablePassthrough — 原样传递

作用： 把输入原封不动地传下去，不修改。

典型用途： 保留原始输入，同时获取模型处理结果。

    flowchart TB
    IN["输入: concept=GIL"]
    IN --> PT[RunnablePassthrough]
    IN --> CH[prompt - llm - parser]

    PT --> OUT1["original: concept=GIL"]
    CH --> OUT2["explanation: GIL是..."]

    style PT fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333
    style CH fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333

5.2 RunnableLambda — 包装任意函数

作用： 把普通 Python 函数包装成 Runnable，接入管道。

实际用例：添加日志

实际用例：格式转换

5.3 RunnableParallel — 并行分支

作用： 一个输入，同时跑多个分支，结果合并成字典。

数据流：

    flowchart TB
    IN["输入: concept=Python装饰器"]
    IN --> A["分支1: explain_chain"]
    IN --> B["分支2: example_chain"]

    A --> C["explanation: 装饰器是..."]
    B --> D["example: def my_decorator..."]

    C --> OUT["结果 dict"]
    D --> OUT

    style IN fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333
    style A fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333
    style B fill:#ffd1d1,stroke:#e57373,stroke-width:2px,color:#333
    style OUT fill:#e6f7e6,stroke:#4caf50,stroke-width:2px,color:#333

简写语法： | 右边直接传字典

5.4 三个工具类对比

工具类 作用 输入 → 输出 适用场景
RunnablePassthrough 原样传递 x → x 保留原始输入
RunnableLambda 包装函数 x → f(x) 自定义处理、日志、格式转换
RunnableParallel 并行分支 x → {k1: f1(x), k2: f2(x)} 多角度分析、同时获取多种结果

6. 高级编排模式

6.1 with_fallbacks — 降级方案

操作速查

操作 写法 说明

Runnable 类型速查

Runnable 输入 输出 用途
