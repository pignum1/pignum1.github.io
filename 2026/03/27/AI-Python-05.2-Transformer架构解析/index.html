AI-Python-05.2: Transformer 架构解析

Transformer 是 2017 年 Google 在论文《Attention Is All You Need》中提出的架构，彻底改变了 NLP 领域。理解 Transformer 是掌握 GPT、BERT、LLaMA 等现代大模型的基础。

1. 为什么需要 Transformer？

1.1 传统序列模型的问题

在 Transformer 出现之前，处理序列数据主要依赖 RNN 和 CNN：

模型 问题
RNN 串行计算无法并行、长距离依赖信息丢失、梯度消失/爆炸
CNN 局部感受野、需要深层堆叠才能扩大感受野、长距离依赖难以建模

1.2 Transformer 的优势

特性 Transformer RNN CNN
并行计算 ✅ 所有位置同时处理 ❌ 串行 ✅ 并行
长距离依赖 ✅ O(1) 直接建模 ❌ O(n) 逐步传递 ❌ 需要深层
位置感知 需要位置编码 ✅ 天然 ✅ 局部

2.3 架构组成

Encoder (左侧): 由 N=6 个相同的层堆叠而成

  • 每层包含: Multi-Head Self-Attention + Feed Forward
  • 每个子层都有残差连接和 Layer Normalization

Decoder (右侧): 同样由 N=6 个相同的层堆叠而成

  • 每层包含: Masked Multi-Head Attention + Cross-Attention + Feed Forward
  • 使用因果掩码防止看到未来信息

3. Self-Attention 详解

3.1 直观理解

想象你在读这句话：

“The animal didn’t cross the street because it was too tired”

当你读到 “it“ 时，你需要知道它指代什么。Self-Attention 让模型学会：

  • “it” 与 “animal” 的关联度很高
  • “it” 与 “street” 的关联度较低

3.3 Q、K、V 的含义

向量 含义 比喻理解
Query (Q) 查询向量 “我在找什么？”
Key (K) 键向量 “我是什么？”（用于匹配）
Value (V) 值向量 “我有什么信息？”（实际信息）

类比图书馆:

  • Query = 你的搜索需求
  • Key = 书籍的分类标签
  • Value = 书籍的实际内容

3.4 计算步骤

步骤1: 生成 Q, K, V

步骤2: 计算注意力分数

步骤3: 缩放

步骤4: Softmax 归一化

步骤5: 加权求和

3.5 完整公式

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$

4.3 为什么需要多头?

单头注意力的局限:

  • 只能学习一种注意力模式
  • 不同位置可能需要关注不同类型的信息

多头注意力的解决方案:

  • 头1 可能关注语法关系
  • 头2 可能关注语义关系
  • 头3 可能关注位置关系
  • …

类似 CNN 的多通道机制，不同头学习不同的信息表示。

7.3 为什么需要 FFN？

Self-Attention 做的事情是聚合信息（看看别人在说什么），但聚合后的信息还需要进一步处理。

FFN 的作用：

  • 对聚合后的信息进行非线性变换
  • 学习更丰富的特征表示
  • 增加模型的表达能力

类比理解：

7.4 FFN 计算过程

公式：
$$
\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2
$$

关键参数：

参数 值 说明
d_model 512 输入/输出维度
d_ff 2048 中间维度 (4×d_model)
激活函数 ReLU / GELU GELU在现代LLM更常用

7.5 FFN 的特点

1. 位置独立 (Position-wise)

  • 每个位置的向量独立通过同一个 FFN
  • 不同位置之间没有交互（交互在 Attention 中完成）

2. 非线性是关键

  • 如果没有激活函数，两层 Linear 等价于一层 Linear
  • ReLU/GELU 引入非线性，让模型能学习复杂函数

8. Layer Normalization

8.1 Layer Norm 图示


Layer Normalization 图示

8.2 为什么需要 Layer Norm？

深层网络训练容易出现内部协变量偏移问题，Layer Norm 可以：

  • 稳定每层的输入分布
  • 加速训练收敛
  • 防止梯度消失/爆炸

8.3 Layer Norm 公式

$$
\text{LayerNorm}(x) = \gamma \cdot \frac{x - \mu}{\sigma} + \beta
$$

其中：

  • $\mu$ = 均值
  • $\sigma$ = 标准差
  • $\gamma, \beta$ = 可学习参数

8.4 Batch Norm vs Layer Norm

对比 Batch Norm Layer Norm
归一化维度 batch 维度 特征维度
适用场景 CV (图像) NLP (序列)
对 batch size 敏感 是 否
推理时 需要记录统计量 不需要

9. 位置编码 (Positional Encoding)

9.1 为什么需要位置编码?

Self-Attention 是位置无关的，输入顺序变化不影响结果。但语言顺序很重要（”狗咬人” vs “人咬狗”）。

9.2 正弦位置编码公式

$$
\begin{align}
PE_{(pos, 2i)} &= \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right) \
PE_{(pos, 2i+1)} &= \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
\end{align}
$$

  • 偶数维度用 sin，奇数维度用 cos
  • 不同频率的正弦波组合
  • 可以外推到训练时未见过的长度

9.3 三种位置编码对比

类型 代表模型 优点 缺点
学习式 BERT, GPT-2 简单效果好 无法外推
正弦式 原始Transformer 可外推 非学习
RoPE LLaMA, GPT-NeoX 可外推+相对位置 -

11. 推理优化: KV Cache

11.1 为什么需要 KV Cache?

自回归生成过程:

问题: 每次生成都要重新计算所有之前 token 的 K, V，计算量 O(n²)

解决方案: 缓存已计算过的 K, V，每次只计算新 token，计算量降为 O(n)

总结

核心公式

$$
\begin{align}
\text{Attention}(Q, K, V) &= \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V \[8pt]
\text{MultiHead}(Q,K,V) &= \text{Concat}(\text{head}_1,…,\text{head}_h)W^O \[8pt]
\text{FFN}(x) &= \text{ReLU}(xW_1 + b_1)W_2 + b_2
\end{align}
$$

关键技术点

技术 作用
Self-Attention 建模全局依赖
Multi-Head 学习多种模式
Positional Encoding 注入位置信息
Layer Norm 稳定训练
Residual 梯度流动
KV Cache 优化生成效率
