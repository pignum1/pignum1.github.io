总结

GIL 影响对比

任务类型 GIL 影响 推荐方案
CPU 密集型 严重瓶颈 多进程 (multiprocessing)
I/O 密集型 影响小 多线程 (threading)
混合型 中等 线程池 + 进程池

线程同步工具

工具 说明 使用场景
Lock 互斥锁 保护共享资源
RLock 可重入锁 同一线程多次获取锁
Event 事件 简单的通知机制
Condition 条件变量 生产者-消费者
Semaphore 信号量 限制并发数量
Barrier 屏障 等待多个线程同步

与 Java 对比

特性 Python Java
全局锁 GIL 无
线程创建 threading.Thread new Thread()
线程池 ThreadPoolExecutor ExecutorService
锁 threading.Lock ReentrantLock
同步块 with lock: synchronized

AI 开发最佳实践

  1. I/O 密集型用多线程: 数据加载、API 调用
  2. CPU 密集型用多进程: 模型训练、数据处理
  3. 使用线程池: ThreadPoolExecutor 更简洁
  4. 注意线程安全: 共享资源加锁
  5. 避免死锁: 按顺序获取锁
