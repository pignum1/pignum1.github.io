总结

ABC 核心组件

组件 说明
ABC 抽象基类，继承自此的类不能直接实例化
@abstractmethod 标记抽象方法，子类必须实现
@abstractproperty 标记抽象属性（已弃用，用 @property + @abstractmethod）
@abstractclassmethod 标记抽象类方法（已弃用）
@abstractstaticmethod 标记抽象静态方法（已弃用）
__subclasshook__ 自定义子类检查逻辑
register() 注册虚拟子类

ABC vs Java 对比

特性 Python ABC Java
抽象类 class X(ABC) abstract class X
抽象方法 @abstractmethod abstract
接口 多重 ABC interface
默认方法 普通方法即可 default
多继承 支持 不支持，用接口

AI 框架中的典型应用

  • PyTorch: nn.Module 基类
  • TensorFlow: tf.Module 基类
  • 插件系统: 插件接口定义
  • 数据处理: Dataset/DataLoader 协议

设计原则

  1. 接口优先: 优先定义 ABC 接口
  2. 组合优于继承: 使用 Mixin 模式
  3. 鸭子类型: Python 风格，__subclasshook__
  4. 最小化抽象: 只抽象必要的方法
