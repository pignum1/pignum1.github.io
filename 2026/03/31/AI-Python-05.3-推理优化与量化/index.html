AI-Python-05.3: 推理优化与量化

大模型推理优化的目标是：更快、更省、更长。更快指的是推理速度，更省指的是显存占用，更长指的是支持的上下文长度。

1. 为什么需要推理优化？

1.1 大模型推理的瓶颈

瓶颈 问题描述
显存占用大 70B模型需要140GB显存（FP16）
推理速度慢 生成一个Token需要几百毫秒
长上下文难 KV Cache随序列长度线性增长
带宽瓶颈 GPU计算快，但显存读写慢

1.2 优化的三个维度

方向 关键技术
模型压缩 量化/剪枝/蒸馏
计算优化 FlashAttention / 投机解码
内存优化 KV Cache / PagedAttention

2. KV Cache 详解

2.1 什么是 KV Cache？

在自回归生成中，每生成一个新Token都需要重新计算所有之前Token的K和V向量。KV Cache通过缓存这些向量避免重复计算。

2.2 KV Cache 的显存占用

以 LLaMA-7B 为例（num_layers=32, num_heads=32, head_dim=128, FP16）：

  • 每 Token 的 KV Cache = 2 × 32 × 32 × 128 × 2 = 512KB
  • 2048 Token 上下文 ≈ 1GB
  • 4096 Token 上下文 ≈ 2GB

3. FlashAttention：计算优化

3.1 标准Attention的问题

标准Self-Attention需要将完整的注意力矩阵存储到HBM（高带宽内存）中：

问题：O(N²) 的显存占用 + HBM读写成为瓶颈（HBM带宽约2TB/s，SRAM带宽约19TB/s）

3.2 FlashAttention 的解决方案

核心思想：分块计算（Tiling）+ 在SRAM中完成所有计算

  1. 将 Q, K, V 分成小块（fit in SRAM）
  2. 在 SRAM 中计算局部注意力
  3. 使用在线 Softmax 技巧合并结果
  4. 避免在 HBM 中存储完整的 N×N 注意力矩阵

优势：显存从 O(N²) 降到 O(N)，计算速度提升 2-4x

4. PagedAttention：内存优化

4.1 KV Cache 的内存碎片问题

传统 KV Cache 的三大问题：

  1. 预分配固定长度 → 浪费内存（短文本）
  2. 动态增长 → 内存碎片化
  3. 多请求并行 → 每个请求独立的连续内存块

现有系统浪费 60% - 80% 的内存！

4.2 PagedAttention（vLLM）

核心思想：借鉴操作系统的虚拟内存分页机制

  • 将 KV Cache 划分为固定大小的”页”（Block）
  • 按需分配，不预分配
  • 非连续存储，逻辑上连续

4.3 PagedAttention 优势

指标 传统KV Cache PagedAttention
内存利用率 20-40% >95%
内存碎片 严重 几乎无
并发能力 受限于预分配 按需扩展
共享前缀 需要复制 天然支持

5. 模型量化

5.1 什么是量化？

量化 = 降低数值精度以减少存储和计算开销

精度 每参数字节 效果
FP16 (16位浮点) 2 字节 基准
INT8 (8位整数) 1 字节 显存减半
INT4 (4位整数) 0.5 字节 显存降到 1/4

5.2 量化的类型

类型 描述 优点 缺点
PTQ (训练后量化) 直接量化训练好的模型 简单快速 精度损失较大
QAT (量化感知训练) 训练时模拟量化效果 精度损失小 需要重新训练
GPTQ 基于二阶信息的PTQ 平衡精度和速度 需要校准数据
AWQ 激活感知量化 保护重要通道 需要校准数据

5.3 量化效果对比

以 LLaMA-7B 为例：

精度 模型大小 显存需求 精度损失
FP16 13.5GB ~28GB 0%
INT8 6.7GB ~14GB <1%
INT4 3.4GB ~7GB 1-3%
INT2 1.7GB ~4GB 5-10%

6. 投机解码（Speculative Decoding）

6.1 原理

核心思想：用小模型”猜测”多个Token，大模型并行验证

投机解码流程：

  1. 小模型快速生成 K 个候选 Token
  2. 大模型一次性验证这 K 个 Token
  3. 接受正确的，拒绝错误的
  4. 从第一个错误的位置重新生成

优势：小模型生成速度快 + 大模型并行验证多个 Token → 整体吞吐量提升 2-3x
